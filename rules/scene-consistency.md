## 一、俯拍图（Top-Down View）

鸟瞰场景全貌，从正上方垂直向下看。

**最低要求**：
```
□ 完整空间边界清晰可见（墙壁/围栏/地形边缘）
□ 所有出入口位置可见（门/通道/楼梯口）
□ 主要建筑/结构的位置和比例准确
□ 角色可行走路径清晰（动线可见）
□ 地面材质/纹理可辨识
□ 正上方垂直视角（非倾斜鸟瞰）
□ 附方向标注（N/S/E/W 或前后左右）
```

**生成 prompt 模板**：
```
[场景描述] 的俯拍图，鸟瞰视角，正上方垂直向下。
展示完整空间布局：边界（墙壁/围栏）、所有出入口（门/通道）、
主要建筑/结构位置、地面材质、角色动线。
附方向标注（前后左右）。比例准确、空间关系清楚。
```

**优点**：空间布局一目了然，适合固定场景
**缺点**：缺少立面信息，无法看到墙壁/高度
**适用**：室内房间、庭院、城镇布局、战场地形

## 二、4 宫格（4-Grid）

场景 4 个关键角度拼为一张。

**4 格强制内容**：

| 格 | 内容 | 锁定维度 |
|----|------|---------|
| 左上 | 正面全景 | 主视角，展示空间纵深+主光源方向 |
| 右上 | 侧面全景（左或右） | 空间侧边界+侧立面 |
| 左下 | 背面全景 | 空间反方向结构+背面光源 |
| 右下 | 关键地标/道具近景 | 核心辨识元素+空间比例参照 |

**最低要求**：
```
□ 4 格覆盖不同方向（禁止4格都是正面略偏不同角度）
□ 光线/色调/天气跨格一致
□ 空间关系可还原（看完4格能定位主要结构和方向）
□ 右下格必须包含比例参照（人物剪影或标准物体）
```

**生成 prompt 模板**：
```
4 宫格空间锁定参考板，同一[场景名称]的 4 个角度：
左上：正面全景（完整空间纵深+主光源方向）
右上：侧面全景（左侧或右侧空间边界）
左下：背面全景（空间反方向）
右下：核心地标/关键道具近景（含角色尺度参照）
光线/色调/氛围一致，禁止跳色跳光，禁止4格都是相似角度。
```

**优点**：快速覆盖主要方向
**缺点**：角度有限，复杂场景不够
**适用**：中小场景（房间/庭院/走廊/街道）

## 三、9 宫格（9-Grid）— 空间锁定

> ⚠ 九宫格是 **🔒 consistency_asset**（空间一致性锚点），不是氛围拼贴。9 格必须覆盖全部空间维度，禁止 9 格都是"好看的相似角度"。

**9 格强制内容**（见 `templates/scene-card.md` → LS9）：

| 格 | 内容 | 锁定的维度 |
|----|------|----------|
| 1 | 正面全景 | 完整空间纵深、主光源方向、天空/天花板 |
| 2 | 左侧全景 | 左侧空间边界和结构 |
| 3 | 右侧全景 | 右侧空间边界和结构 |
| 4 | 入口/起点 | 空间进入点、通道位置 |
| 5 | 核心地标 | 最具辨识度的空间元素（建筑/巨物/神像） |
| 6 | 关键道具/巨物 | 关键物体与空间的比例关系 |
| 7 | 地面材质 | 地面纹理、反射、磨损（MT编号） |
| 8 | 天空/主光源 | 光源方向、色温、体积光、天空状态 |
| 9 | 角色尺度参考 | 空间尺度参照（角色剪影/身高标记） |

**品质要求**：
```
□ 所有 9 格光线一致、色调统一、天气/时段一致
□ 9 格覆盖不同空间维度（非同一角度的 9 种变体）
□ 建筑风格/材质/比例跨格一致
□ 空间关系可还原（看完 9 格能在脑中重建场景 3D 布局）
```

**禁止事项**：
```
❌ 9 格都是不同角度的"好看远景"（那是氛围拼贴，不是空间锁定）
❌ 9 格像 9 个不同地点
❌ 天气/时段/色温跳变
❌ 文字/标注过多遮挡空间信息
```

**生成 prompt 模板**：
```
3×3 九宫格空间锁定参考板，同一[场景名称]。
格1正面全景：完整空间纵深+主光源方向+天空。格2左侧全景。格3右侧全景。
格4入口/起点。格5核心地标特写。格6关键道具/巨物（含人物比例参照）。
格7地面材质特写。格8天空+主光源方向+色温。格9角色尺度参照。
所有9格光线一致、色调统一、氛围连贯、空间可还原。
```

**优点**：空间全覆盖，复杂场景不遗漏，可重建 3D 空间关系
**缺点**：生成成本较高
**适用**：复杂场景（城镇/战场/宫殿/森林/洞穴）、需要空间一致性的多角度视频

## 四、全能参考模式（All-in-One Reference）

一张图同时锁死场景 + 角色 + 光照 + 道具，效率最高的锚点方案。

**最低要求**：
```
□ 场景全景为主体背景（占画面 ≥ 60%）
□ 角色站立于场景中（占画面 20-30%），展示全身+服装+姿态
□ 主光源方向明确可见（箭头或光影指示）
□ 地面材质可辨识
□ 关键道具/元素位置清楚（与角色的空间关系明确）
□ 天空/天花板色调确定
□ 同一张图中场景+角色+光+道具必须来自同一世界观（禁止拼贴风）
```

**生成 prompt 模板**：
```
全能参考图：[场景名称]全景环境为主体，[角色名]站立于场景中央偏左，
面向右方呈标准站姿，完整展示面部/发型/服装/配饰/武器（如有）。
场景中清晰标注：主光方向（箭头+色温）、地面材质、天空色调、
关键道具位置（红框标注）。整体氛围为[时间/天气/情绪]。
角色占画面 1/3，场景占 2/3。
```

**布局结构**：
```
┌─────────────────────────────────┐
│        场景全景环境              │
│  ☀ 主光方向 →                   │
│    ┌────┐                       │
│    │角色│  地面材质              │
│    │站姿│  ▓▓▓▓▓▓▓▓            │
│    └────┘                       │
│         □ 关键道具（红框标注）    │
│         天空色调：xxxx            │
└─────────────────────────────────┘
```

**优点**：
- 一张图锁定全部变量，Seedance 只需 1 张参考图 + prompt
- 场景/角色/光照/道具同框，AI 自动学会关联关系
- 适合快速验证、成本最低

**缺点**：
- 角色细节不如独立角色卡丰富
- 换场景需重新生成

**适用**：快速出片、单场景视频、Seedance 简略模式

**触发命令**：回复「全能参考图」或「all-in-one」

## 五、720° 全景图（720° Panorama）

生成或合成一张 360°×180° 的全景环境图，覆盖所有可能拍摄方向。

**6 面最低要求**：

| 面 | 方向 | 必须包含 |
|----|------|---------|
| 1 | 前（Front） | 场景主视角全景，入口/主建筑/核心地标 |
| 2 | 后（Back） | 反方向全景，后墙/反方向出口/背面结构 |
| 3 | 左（Left） | 左侧空间边界+左侧立面结构 |
| 4 | 右（Right） | 右侧空间边界+右侧立面结构 |
| 5 | 上（Up） | 天空/天花板+主光源方向+体积光 |
| 6 | 下（Down） | 地面全貌+地面材质+角色站立区域 |

```
□ 6 面之间边缘可无缝拼接（重叠区域 ≥ 10%）
□ 6 面光线一致、色调统一、同一时段
□ 拼接后无接缝/错位/形变
□ 覆盖 360°×180° 全部拍摄方向
```

**方式 A：多角度合成（推荐）**
- 生成 6 个方向：前/后/左/右/上/下（cubemap 立方体贴图方式）
- 用工具拼接为一张 equirectangular 全景图
- 输入 Seedance 作为全局环境参考
- 出图量：6 张。费用：~$0.24-0.48。耗时：~90s

**方式 B：实景拍摄（最准确）**
- 用 360° 相机（Insta360/Ricoh Theta）或手机全景模式拍摄
- 作为参考图直接输入，无需拼接
- 出图量：1 张。费用：零。耗时：拍摄~30s

**方式 C：直接生成（目前受限）**
- 目前主流模型（DALL-E 3、Flux、SD）**不支持直接生成 equirectangular 全景图**
- 如未来有支持全景的模型，可用 prompt：
  ```
  equirectangular panorama，360 degree，[场景描述]，
  覆盖东西南北四个方向，统一光照和色调，天空完整、地面连续、无接缝
  ```
- 出图量：1 张。费用：取决于模型。耗时：~20s（待模型支持）

**触发命令**：回复「生成全景图」或「720全景」

**优点**：覆盖所有角度，视频中无论镜头转到哪个方向都有参考
**缺点**：方式A需多图拼接，方式B需硬件，大文件

## 六、环绕视频截图法（Orbit Video Screenshot）

先生成一个环绕场景的短视频（或用 3D 引擎渲染），再从中截取关键帧作为多角度参考。

**步骤**：
```
1. 用 AI 视频工具生成一段「环绕场景旋转」的短片（10-15s，保证旋转不急促）
   prompt：「camera orbits 360° around [scene]，steady slow rotation，bird eye view slowly descending to eye level」
   注意：5s 转 360°太快会导致截图模糊，建议至少 10s

2. 截取 6-8 张关键帧（每 45°~60° 一张）

3. 拼接为一张参考图（按角度排列），或单独作为多张参考图输入
```

**使用 Seedance 生成环绕视频**：
```
prompt：[场景名称]，camera orbits 360 degrees around the scene，
steady slow rotation，no speed change，maintain constant distance，
展示场景完整空间结构
```

**截取角度建议**：
| 角度 | 说明 |
|------|------|
| 0° | 正面 |
| 45° | 右前 |
| 90° | 右侧 |
| 135° | 右后 |
| 180° | 背面 |
| 225° | 左后 |
| 270° | 左侧 |
| 315° | 左前 |

**优点**：真实连续空间参考，最接近实拍效果
**缺点**：需两轮生成（先视频后截图），耗时加倍
**适用**：3D 引擎场景、实景拍摄、对空间一致性要求极高的项目

**触发命令**：回复「环绕截图」或「视频截图法」

## 七、文字场景锁定 · 多角度分镜（Text-Based Scene Lock）

通过精确文字描述锁定同一场景的布局/道具/光线，逐镜从不同角度生成分镜图。

**核心思路**：不依赖图像参考（DALL-E 3 / GPT Image 2 不支持 img2img），
而是用**场景说明书**（一段精确的场景文字描述）作为每镜 prompt 的共同前缀，
保证 AI 每一镜都「看到」同样的场景。

**步骤**：
```
1. 先写一份「场景说明书」（200-500 字的场景精确描述）
   → 包含：空间尺寸、建筑/家具位置、门窗方向、主光源位置、
           色温、材质、天空/地面颜色、关键道具位置
   → 这段文字是后续所有镜头的共同前缀，一字不改

2. 每镜 prompt = 场景说明书 + 镜头指令：
   → 镜 1：[场景说明书] + 「从门口看向室内，低角度仰拍，突出空间高度」
   → 镜 2：[场景说明书] + 「从窗口看向角色位置，侧面中景，柔光逆光」
   → 镜 3：[场景说明书] + 「从天花板角落俯拍，广角，覆盖整个空间」
   → 镜 N：[场景说明书] + 「角色 POV 视角，看向目标方向」
```

**场景说明书模板**：
```
[场景名]：一间 6m×8m 的中式书房。东墙是整面书架，西墙有一扇圆形漏窗，
南面是一张红木书桌（2m×1m，靠南墙正中），北面入口是一扇双开木门。
主光源：南窗自然光（色温 5600K，从南向北斜射，角度约 45°）。
辅助光源：书桌上一盏铜质台灯（暖黄 3200K）。
地面：深色木地板，有磨损痕迹。天花板：木梁裸露，高 3.5m。
色调：暖棕+墨绿。氛围：午后安静书房，阳光透过窗格投下条纹光影。
```

**优点**：
- 不依赖 img2img 能力，任何文生图模型都可用
- 场景说明书可复用，换镜头只需改后半句
- 每镜都是独立高质量图

**缺点**：
- 场景一致性依赖文字描述精度，不如图像参考直观
- 复杂场景说明书本身需要仔细写
- 不同模型对同一文字的理解有差异

**触发命令**：回复「镜头角度分镜」或「camera remix」

**真正基于图像参考的多角度分镜**（需要 img2img 能力）：

如果需要真正「基于一张图改角度」，目前可行的方案：

| 方案 | 工具 | 做法 |
|------|------|------|
| ControlNet Canny | ComfyUI + SD | 场景图提取边缘 → 固定构图 → 改 prompt 换角度 |
| ControlNet Depth | ComfyUI + SD | 场景图提取深度 → 保持空间结构 → 换视角 |
| IP-Adapter | ComfyUI + SD | 场景图作为风格/内容参考 → 换构图 |
| Nano Banana | Gemini API | 14 张参考图输入 → 多轮对话改角度（真正支持） |

**与其他方法配合**：
```
最强组合：720° 全景图 + 文字场景锁定
1. 先生成 720° 全景图（用 6 面 cubemap 拼接）
2. 从全景图中提取场景说明书（肉眼观察 + 文字描述）
3. 场景说明书作为共同前缀，逐镜生成各角度分镜图
4. 得到场景一致的 N 张分镜图

如果使用 ComfyUI + ControlNet Depth：
1. 700° 全景图 → 提取深度图 → 每个镜头角度用 3D 投影生成对应视角的深度图
2. 用对应视角深度图 + prompt → 生成该角度分镜图
3. 这是目前技术上最强的场景一致性方案
```
