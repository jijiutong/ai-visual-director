## 一、角色设定卡（6 模块）

一张图包含 6 个模块，全面锁定角色外观。

**6 模块布局**：
```
┌──────────────┬──────────────┬──────────────┐
│  模块1：三视图  │ 模块2：面部    │ 模块3：表情    │
│  正面/侧面/背面 │ 正面/45°/侧面  │ 平静/愤怒/笑   │
│  全身站姿      │ 特写          │ 悲伤/惊讶/厌恶  │
├──────────────┼──────────────┼──────────────┤
│  模块4：手部    │ 模块5：武器/道具 │ 模块6：发型动态  │
│  正面/背面     │ 正面+侧面特写   │ 静止/飘动/扎起  │
│  握拳/展开     │ 红标不可变细节   │ 红标不可变细节  │
└──────────────┴──────────────┴──────────────┘
```

**生成 prompt 模板**：
```
6 模块角色设定卡，[角色名]：[性别/年龄/身份]。
模块1 左列：三视图（正面全身站姿/右侧面/背面），[服装描述]。
模块2 中上：面部多角度（正面/45度/正侧面特写），[发型/五官描述]。
模块3 右上：6 种表情（平静/愤怒/大笑/悲伤/惊讶/厌恶），表情自然不过度。
模块4 左下：手部特写（正面展开/背面握拳），手指比例正常，[手部特征]。
模块5 中下：关键道具/武器正面+侧面，以视觉方式突出不可变细节（用高光/对焦/放大局部引导视线，而非文字标注）。
模块6 右下：发型动态（静止/微风飘动/扎起），以视觉方式突出发色/发量/发长。
白色/浅灰背景，均匀摄影棚光，禁止彩色背景。6 格白线分隔。
角色服装/发色/五官/体型 6 格完全一致，禁止跨格变化。
禁止在画面中出现：红色标注线、箭头、文字、编号、方框、引线。
所有细节通过光影、对焦、构图对比传达，不叠加任何文字或符号。
```

**作用**：一张图锁定角色在 AI 视频中的所有变量。Seedance/Runway/可灵均可直接作为参考图输入。

**触发命令**：回复「生成角色卡」或「格式3」

## 二、三视图（正/侧/背）

最精简的角色一致性方案，仅锁形体。

**生成 prompt 模板**：
```
[角色名]三视图：正面全身站姿（居中）、右侧面（右）、背面（左）。
三视图并排，同一角色，服装/体型/发色/鞋履完全一致。
白色背景，均匀摄影棚光。站姿自然放松，手臂自然下垂。
[角色详细描述]
```

**优点**：简单、出图快、任何平台都支持
**缺点**：缺少面部细节和表情参考
**适合**：快速验证、作为辅助参考

## 三、面部多角度（5 角度）

专攻面部一致性，5 个角度确保 AI 理解角色的面部结构。

**5 角度布局**：
```
正面 → 右 45° → 右侧面 → 左 45° → 左侧面
```

**生成 prompt 模板**：
```
[角色名]面部多角度参考：正面、右45度、正右侧面、左45度、左侧面。
5 张面部特写并排，肩部以上。同一角色，五官/肤色/发型/发色/瞳色完全一致。
[五官描述]：眼型/鼻型/嘴型/眉型/脸型。
中性表情，均匀柔光，浅灰背景。禁止跨角度五官变形。
```

**优点**：面部一致性极强，AI 视频工具面部变形率大幅降低
**缺点**：不含全身/服装信息

## 四、表情范围图（12 表情）

覆盖角色全情绪范围，防止 AI 在情绪表达时面部走样。

**生成 prompt 模板**：
```
[角色名]表情范围图：3×4 宫格，12 种表情。
上排：平静 / 微笑 / 大笑 / 愤怒
中排：悲伤 / 哭泣 / 惊讶 / 恐惧
下排：厌恶 / 轻蔑 / 害羞 / 坚定
所有表情同一角色，五官/肤色/发色/瞳色完全一致。
肩部以上特写，白色背景，均匀柔光。
中性表情（左上第一格）作为面部基准。
```

**优点**：锁定情绪表达范围，防止表情变化时角色变脸
**缺点**：不含身体/服装

## 五、服装/武器细节卡

锁定服装和武器的不可变细节。

**生成 prompt 模板（出图模式 · 默认）**：
```
[角色名]服装武器细节参考卡。
左半：服装正面全身+背面，以视觉方式突出不可变细节（面料材质/颜色/纹样/配饰位置/腰带扣形制），用高光与对比度引导视线。
右半：武器正面+侧面+细节特写，以视觉方式突出不可变细节（纹路/比例/材质/颜色/握柄形制）。
白色背景，均匀光。服装和武器各占一半，清晰分区。
禁止红色标注、文字、箭头、编号。
```

**导演标注版**（仅当用户明确说「导演标注版」「标注」时）：允许红色标注不可变细节。

**作用**：补充角色卡中可能被压缩的服装/道具细节，独立参考图确保不丢失。

## 六、Nano Banana 14 图参考（最强文字+图像锚定）

Google Gemini 的 Nano Banana 支持最多 14 张参考图输入，是当前人物一致性最强的方案。

**步骤**：
```
1. 准备 12-14 张角色参考图：
   · 三视图（3 张：正/侧/背）
   · 面部 5 角度（5 张）
   · 表情 4 张（喜怒哀惧）
   · 服装细节（1-2 张可选）
   
2. 全部输入 Nano Banana，prompt：
   「基于以上参考图，生成[角色名]在[场景]中的[动作]，保持角色完全一致」

3. 支持多轮对话：
   「让他转过来」→ 同角色背面
   「换个角度从右边看」→ 同角色不同角度
```

**费用**：Pro $0.24/4K 张，或免费额度

**优点**：14 张参考图直接输入，AI 真正"看到"角色全貌。支持多轮对话改角度。中文原生。
**缺点**：仅 Nano Banana 支持，其他平台无此能力。14 张参考图准备成本高。

**触发命令**：回复「Nano Banana 角色锚定」或「14 图参考」

## 七、ComfyUI IP-Adapter（本地最强）

本地 SD/Flux + IP-Adapter，零成本、无限生成。

**步骤**：
```
1. 准备一张高质量角色参考图（正面全身即可）
2. 在 ComfyUI 中加载 IP-Adapter 节点
3. IP-Adapter 将角色参考图编码为特征向量
4. 后续所有生成都注入该向量，角色保持一致
```

**ComfyUI 节点链**：
```
Load Image (角色参考) → CLIP Vision Encode → IP-Adapter Apply
                                                      ↓
Prompt → CLIP Text Encode → KSampler → VAE Decode → Output
```

**优点**：零成本、无限生成、角色一致性极强、结合 ControlNet 可约束姿势
**缺点**：需要本地 GPU + ComfyUI 部署（门槛高）、学习曲线

**触发命令**：回复「IP-Adapter 角色锁定」

## 八、角色 DNA 20 字段（纯文字锚定）

不生成图片，而是在所有 prompt 中嵌入统一的角色文字描述。

**20 字段**：
```
1. 姓名    2. 性别年龄    3. 身高体型    4. 脸型    5. 眼型瞳色
6. 眉型    7. 鼻型        8. 嘴型唇色    9. 发型发色  10. 肤色
11. 服装   12. 配饰        13. 武器/道具   14. 声音     15. 体态站姿
16. 习惯动作 17. 气味      18. 情绪底色    19. 可变字段  20. 不变量标注
```

**使用方式**：每镜 prompt 共用同一段 DNA 描述作前缀，确保文字层面一致。

**可变字段（追踪变化）**：
```
· 服装：镜1-3 战甲 → 镜4-7 便装（战损后更换）
· 发型：镜1-4 束发 → 镜5-7 散发（战斗中散开）
· 情绪：按情绪曲线逐镜变化
```

**作用**：文字层面保底，与图像参考叠加使用效果最佳。

**触发命令**：回复「角色 DNA」或「DNA 锚定」
