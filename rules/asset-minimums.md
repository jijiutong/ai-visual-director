# 最低资产包规则

把 `consistency-trigger` 的建议升级为部分强制规则。不同项目风险对应不同最低资产包。高风险项目不满足最低资产时阻断进入 video prompt assembly。

---

## 一、资产包等级

| 等级 | 资产 | 出图量 | 适用场景 |
|------|------|--------|---------|
| **快速包** | 角色卡 + 场景卡 + 分镜图 | 3-5张 | 一句话短视频（≤3镜），快速验证 |
| **标准包** | 角色卡 + 场景卡 + 分镜图 + 首帧 | 5-7张 | 正常 10-15s 视频，单场景单主角 |
| **正式包** | 角色卡 + 面部多角度 + 服饰细节 + 场景九宫格 + 分镜图 + 首尾帧 | 8-12张 | 主角多镜、复杂服装、多角度场景 |
| **长篇包** | 正式包 + continuity-snapshot + project-graph | 12+张 | 多段/多章/系列 |
| **展示包** | 全案板 + 角色研究板 + 场景板 + 海报 | 6-10张 | 提案和团队沟通（不要求视频锚点强度） |

---

## 二、强制升级规则

以下条件触发时，`asset-plan` **必须**升级资产包，不可降级。`consistency-trigger` 检测到不满足时**阻断**主链。

| 条件 | 强制资产 | 升级原因 |
|------|---------|---------|
| 主角 > 4 镜 | 角色卡 + 面部多角度（至少方法3） | 多镜出镜面部易漂移 |
| 主角 > 4 镜 且 复杂服装（多层/铠甲/纱衣） | + 服饰细节卡（方法5 或 LS44） | 服装结构复杂跨镜易变形 |
| 长发/纱衣/发饰复杂 | LS42 仙灵长发板 或 LS44 服饰细节留白板 | 长发和纱衣是AI最易漂移的元素 |
| 双主角关系戏 | LS24 角色关系双人板 | 双人空间关系需锚定 |
| 多场景视频（≥2场景） | 每个主场景至少一张场景卡 | 跨场景空间易漂移 |
| 同一场景多角度（仰+平+俯全含） | LS9 九宫格 或 LS27 蓝图俯视 | 多角度场景需全局锚定 |
| 多段视频（segment_count ≥ 2） | continuity-snapshot 必须写入 + 每段独立全案板 | 跨段连续性保障 |
| 平台参考图少（≤3张） | prompt DNA 必须增强（角色DNA 20字段 + 场景DNA 文字描述） | 参考图不足靠文字补偿 |
| 武器/道具角色 | 武器/道具细节卡 | 道具形制跨镜保持一致 |
| 变身/战损/进化剧情 | 面部12表情（方法4）+ 各形态角色卡 | 角色形态变化需要多重锚定 |

## 三、快速包适用条件（可降级的例外）

仅以下条件全部满足时可以使用快速包：

```
□ 镜数 ≤ 3镜
□ 单角色（或最多1配角）
□ 单场景
□ 简单服装（单层，无特殊材质）
□ 无特殊武器/道具
□ 正面为主拍摄角度
□ 不是视频用途（仅概念验证）
```

任一不满足 → 至少升级到标准包。

## 四、阻断机制

`consistency-trigger` 在建议模式基础上增加强制检查：

```
1. 读取 asset-plan 的当前资产列表
2. 对照本规则强制升级条件
3. 如触发强制条件但资产不满足：
   → ⚠ 阻断，不进入 reference-anchor
   → 输出缺失资产清单 + 强制升级原因
   → 等待用户确认或自动补充
4. 如不触发强制条件：
   → 保持建议模式（不阻断）
```

阻断输出格式：

```markdown
⚠ 一致性升级为强制：

原因：主角出镜 6 次，服装为多层白纱长袍，发型为超长白发。
触发规则：主角>4镜且复杂服装 + 长发/纱衣复杂

必须补充资产：
  - 面部多角度（方法3）
  - LS42 仙灵长发角色设定板
  - LS44 服饰细节留白板

当前缺失：面部多角度、LS42、LS44
阻断原因：不补充则角色跨镜漂移风险极高。

回复「自动补充」→ 一键生成缺失资产
回复「跳过风险」→ 强制放行（角色一致性不保证）
```
