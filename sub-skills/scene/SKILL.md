# Scene — 场景设计

> **治理**：走 `engines/command-gate.md` 权限表 §四 /avd/scene。产出标记 `draft`，不写回主状态。禁止重设角色脸/续写剧情/音效台词/D类融合。详见 `rules/format-contract.md` §1.2。

输入场景描述 → 选择定位方法 → 输出场景概念图 prompt。7 种场景一致性锁定方法。

## 触发方式

- `/avd/scene [场景描述]`
- 直接说 "场景"、"场景设计"、"场景概念图"、"环境设计"

## ⚠️ 路由规则

| 用户说法 | 行为 |
|---------|------|
| `/avd/scene` / `/avd/scene 场景图`（未指定方法） | **展示 S1-S7 方法表**，等用户选方法后再生成 |
| `/avd/scene S4` / `/avd/scene 全能参考图` | 直接用 S4 生成 |
| `/avd/scene S3` / `/avd/scene 九宫格` | 直接用 S3 生成 |
| `/avd/scene [场景描述]`（有新场景信息） | 收集信息 → 展示方法表 → 用户选 → 生成 |

**禁止**：不展示方法表直接生成。`/avd/character` 有 C1-C8 选单，`/avd/scene` 同理必须先展示 S1-S7 让用户选。

## 子指令

| 指令 | 效果 |
|------|------|
| `/avd/scene 场景锚点` | 自动推荐最适合的锁场景方法 |
| `/avd/scene 全能参考图` / `all-in-one` | 一张图锁定场景+角色+光照+道具+氛围+构图 |
| `/avd/scene 九宫格锁定` | 3×3 多角度场景锁定，适合复杂宫殿/战场/太空基地 |
| `/avd/scene 双联宽幅锚点` | 上下两张 21:9 场景帧，适合视频首尾/阶段锚点 |
| `/avd/scene 720全景` | 360°×180° 全景环境图 |
| `/avd/scene 环绕截图` / `视频截图法` | 环绕视频→截取 8 角度→场景参考板 |
| `/avd/scene camera remix` | 同场景多角度镜头变化，保持空间一致 |

## 工作流

### Step 1: 收集场景信息

从用户输入提取：场景名称、地点类型、空间尺寸、风格时代、主光源、天气、色调、氛围。

### Step 2: 选择场景锁死方法（7 种）

**读取 `rules/scene-consistency.md`** → 获取 7 种方法的最新参数（出图量/费用/锁定强度/适用场景）。方法参数**不在此 SKILL 中写死**，以 `rules/scene-consistency.md` 为权威源。读取后按其中的「方法选择指南」表呈现给用户选择。

以下为简要速查（完整数据以 rules 文件为准，锁定强度/费用/耗时可能已被更新）：

| 编号 | 方法 | 适用 |
|------|------|------|
| **S4** | 全能参考板（LS8）⭐默认 | 15s+ / 单场景全面锁定 |
| S3 | 九宫格空间锁定（LS9） | 复杂场景/宫殿/战场/多角度需求 |
| S2 | 4宫格 | 中小场景/4个关键角度 |
| S1 | 俯拍图（LS27） | 室内/庭院/需锁定空间布局 |
| S10 | 双联宽幅锚点 | 视频首尾帧/双状态对比 |
| S5 | 720°全景 | 30s+/多角度视频拍摄 |
| S7 | 文字场景锁定 | 快速版/零成本保底 |

> 锁定强度/出图量/费用/耗时 **不在此表中写死**——每次从 `rules/scene-consistency.md` 读取最新值。

### Step 3: 生成 Prompt

> **⚠️ 模板强制规则**：生成场景图 Prompt 前，**必须依次读取**：
> 1. `templates/scene-card.md`（Prompt 结构权威源——LS版式决策表、变量编号 EV/HE/WT/MT/CP/CN/SD/SP/CR、附加模块格式）
> 2. `rules/scene-consistency.md`（所选方法的参数约束和 Prompt 模板）
> 
> 不读模板直接写 = bug。本 Skill 不嵌入 Prompt 模板——全部从上述两个文件读取。

## 场景说明书模板（文字场景锁定用）

详见 `rules/scene-consistency.md` §七。简要格式：

```
[场景名]：一间 [尺寸] 的 [风格描述]。
东墙：[描述] | 西墙：[描述] | 南面：[描述] | 北面：[描述]
主光源：[方向+类型+色温]
辅助光源：[描述+色温]
地面：[材质+颜色+磨损] | 天花板：[描述+高度]
色调：[主色+辅色] | 氛围：[时间+天气+情绪]
```

## 方法叠加策略

从 `rules/scene-consistency.md` 获取最新的叠加策略和费用。费用和耗时**不在此写死**——每次从 rules 文件读取最新值。
