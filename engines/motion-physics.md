# 运动物理引擎

视频不是图，最容易坏在动作冲突。确保人物运动 + 镜头运动 + 环境运动三者协调。

---

## 一、运动预算（硬限制）

| 维度 | 预算 | 说明 |
|------|------|------|
| 相机运动 | 最多 1 个 | Static/Slow Push/Crash Zoom → 选1 |
| 主体运动 | 最多 1 个 | 走/跑/跳/转身/打斗 → 选1 |
| 环境运动 | 最多 1 个 | 雨/风/烟雾/碎片 → 选1 |
| **每镜总计** | **≤ 3 个** | |

### 运动冲突检测

生成前自动检查以下冲突，发现后强制修正：

| 冲突类型 | 检测规则 | 修正 |
|---------|---------|------|
| 总运动超限 | 相机 + 主体 + 环境 > 3 | 删最低优先级运动 |
| 双相机运动 | 选了 Push + Pan | 二选一，或改为 Dolly Zoom 合并 |
| 主体×相机矛盾 | 主体跑动 + 镜头静止 | 改为跟拍（Tracking）或主体改走 |
| 速度不匹配 | 主体快跑 + Slow Push | 统一速度：快切匹配或慢推匹配慢走 |
| 环境运动遮盖 | 烟雾+雨+沙尘 | 只保留1个主环境运动，其余降级为"氛围"，不要求可见运动轨迹 |
| 运动空镜 | 主体、相机均静止 + 无环境运动 | 至少补1个运动维度，或标记为静止定帧（需设计时长≤1s） |

---

| 主体运动 | 适合镜头 | 禁止镜头 |
|---------|---------|---------|
| 站立静止 | Static / Slow Push | Handheld（没必要晃） |
| 慢走/散步 | Tracking / Pan | Crash Zoom |
| 跑步/冲刺 | Tracking（侧跟）/ Follow Cam | Static（人跑出画） |
| 跳跃/飞起 | Crane Up / Whip Pan | Static |
| 打斗/格斗 | Handheld / Steadicam | Static / Slow Push |
| 坠落/下沉 | Pull Down / Tilt Down | Crane Up（方向矛盾） |
| 旋转/转身 | Orbit / Whip Pan | Static |
| 漂浮/悬浮 | Slow Push / Static | Handheld（打破漂浮感） |

## 三、环境运动

| 环境 | 强度 | 说明 |
|------|------|------|
| 树叶飘动 | 微 | 增强自然感，不干扰 |
| 雨水下落 | 中 | 主体快→雨更快，主体慢→雨也慢 |
| 烟雾弥漫 | 慢 | 主体穿行其中=意境 |
| 爆炸/碎片 | 极强 | 主体运动被环境压倒（合理） |
| 风沙呼啸 | 强 | 主体必须对抗环境（合理） |
| 人群走动 | 中 | 主体静止→人群流动=孤独感 |

---

## 四、可接受 & 不可接受组合

| ✅ 可接受 | ❌ 不可接受 | 原因 |
|----------|-----------|------|
| Static + 主体转身 + 树叶微动 | Orbit + Push In | 两个相机运动 |
| Slow Push + 主体沉思 + 无环境 | Handheld + Crane Up | 手持不能同时升降 |
| Handheld + 主体奔跑 + 无环境 | Crash Zoom + Slow Pull | 速度方向相反 |
| Tracking + 主体慢走 + 雨水 | 主体狂奔 + 主体静止沉思 | 逻辑矛盾 |
| Crash Zoom + 主体猛扑 + 无环境 | 下雨+沙尘暴+暴雪+浓雾 | 天气不能同时 |

---

## 五、冲突检测

| 冲突 | 检测 | 修正 |
|------|------|------|
| 方向冲突 | 人物← + 镜头←Pan = 跟不上 | 改为 Tracking |
| 强度不匹配 | 人物静止 + Handheld 剧烈晃动 | 改为 Static |
| 多方向混乱 | Crane Up + 人物下坠 + 碎片↑ | 只保留1个主方向 |
| 速度不匹配 | 慢走 + Crash Zoom | 改为 Slow Push |

---

## 六、运动方向含义

| 方向 | 含义 |
|------|------|
| ← 左 | 回归、过去、退后 |
| → 右 | 前进、未来、进取 |
| ↑ 上 | 希望、权力、升华 |
| ↓ 下 | 坠落、失败、死亡 |
| → 向镜头 | 进攻、面对、挑战 |
| ← 离镜头 | 逃避、离开、结束 |

---

## 七、运动优先级（取舍时）

```
1. 主体运动（最重要 — 观众看的是人）
2. 相机运动（次重要 — 引导注意力）
3. 环境运动（最不重要 — 氛围增强）
```

---

## 八、输出

```markdown
【运动物理方案】

每镜运动配对：
  镜1：主体[运动] + 镜头[运动] + 环境[运动] → [协调/冲突→修正]
  镜2：...

冲突标记：
  ⚠ 镜[N]：[冲突] → 修正为[方案]

每镜运动检查：
  □ 相机运动 ≤ 1？
  □ 主体运动 ≤ 1？
  □ 环境运动 ≤ 1？
  □ 运动方向一致？（没有同时←和→）
  □ 运动速度一致？（没有同时极快极慢）
```
