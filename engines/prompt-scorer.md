# Prompt 评分引擎

生成视频 prompt 后自动评分，低于阈值触发 auto-repair。

---

## 一、评分维度

| 维度 | 权重 | 评分核心 |
|------|------|---------|
| 角色一致性 | 25% | DNA 每镜声明？跨镜不可变特征一致？ |
| 场景一致性 | 20% | 光源/色调/空间关系每镜一致？ |
| 动作可执行 | 20% | 运动 ≤ 3个/镜？方向不冲突？ |
| 平台兼容 | 15% | 字数/语言/参考图/时长在限制内？ |
| 连续性 | 10% | 镜间状态继承？有无跳跃？ |
| Prompt 格式 | 10% | 语法正确？无乱码？参数正确？ |

---

## 二、各维度评分规则

### 角色一致性（满分100）
- DNA 每镜声明（全镜检查，非单镜加分）：+20
- 3个不可变特征每镜一致：+30
- 多角色身高差/体型差一致：+20
- 无"换人"现象：+30
- 扣分：每缺1个DNA字段 -10，每次跨镜矛盾 -20

### 场景一致性（满分100）
- 主光源方向每镜声明且一致：+40
- 主色调每镜一致：+30
- 空间关系每镜一致：+30
- 扣分：光源矛盾 -20，色调漂移 -15，空间漂移 -25

### 动作可执行（满分100）
- 每镜运动 ≤ 3个：+40
- 运动方向逐镜连续：+30
- 人物+镜头+环境不冲突：+30
- 扣分：超运动预算 -25/镜，方向矛盾 -20/镜

### 平台兼容（满分100）
- 字数在限制内：+40
- 语言匹配平台：+25
- 参考图在限制内：+20
- 时长在限制内：+15
- 扣分：超字数 -30，语言不匹配 -20，参考图超限 -15

### 连续性（满分100）
- 每镜结束状态已记录：+40
- 下镜起始 = 上镜结束：+35
- 场景切换有转场：+25
- 扣分：状态缺失 -20/镜，跳跃 -30/次
- 详细检查维度见 `rules/continuity-check.md`（时间线/空间逻辑/角色/情绪曲线/光线 5维度）

### Prompt 格式（满分100）
- 无乱码/语法正确：+40
- 负面提示词匹配平台：+30
- 特殊参数正确：+30
- 扣分：乱码 -30，语法错误 -20，参数错误 -25

---

## 三、总分与操作

```
总分 = 角色×0.25 + 场景×0.20 + 动作×0.20 + 平台×0.15 + 连续×0.10 + 格式×0.10
```

| 总分 | 操作 |
|------|------|
| ≥ SCORE_PASS_THRESHOLD（从 api-config.template.env 读取）| 放行 |
| 70 ~ (SCORE_PASS_THRESHOLD - 1) | 标记问题，放行 |
| 60-69 | 触发 auto-repair |
| < 60 | 强制 auto-repair |
| 任一项 < 50 | 该单项强制修复 |

## 四、输出

```markdown
【Prompt 评分】

角色：85 ✅ | 场景：72 ⚠ | 动作：90 ✅
平台：95 ✅ | 连续：68 ⚠ | 格式：88 ✅

总分：83/100 — B 良好

需关注：
  1. 场景 — 镜2光源左侧→镜3顶光，不连续
  2. 连续性 — 镜4→5缺少状态继承
```

## 联动

← 接收 `video-prompt-assembly` 组装完成的 prompt + 所有前序引擎的输出
← 接收 `consistency-engine` 的一致性评估报告（5 维度评分作为补充参考）
→ 评分低 → 触发 `auto-repair`
→ 修复后重评（最多 REPAIR_MAX_ROUNDS 轮，从 api-config.template.env 读取）
