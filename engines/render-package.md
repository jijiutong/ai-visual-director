## 单段 vs 多段

### 单段输出

如果 `shot-budget` 判断"一次生成"，直接输出上述完整包。

### 多段输出

如果 `shot-budget` 判断"需拆段"：

```markdown
═══════════════════════════════
📦 段1/3：[段描述]
  时长：[Ns] / 镜数：[N镜]
  资产：[列表]

  ────────────────────────────
  段1尾帧：[状态快照，8字段 end_state]
    ↓ 衔接技法：[编号]
  段2首帧：[继承段1尾帧全部 8 字段]
═══════════════════════════════
📦 段2/3：[段描述]
  ...
═══════════════════════════════
📦 段3/3：[段描述]
  ...
═══════════════════════════════
```

## 输出顺序优先级

1. **先出资产Prompt** → 让用户先生成参考图
2. **再出分镜图Prompt** → 有了参考图后再生成分镜图
3. **最后出视频Prompt** → 所有参考图就绪后可执行

标记每个步骤的依赖关系：

```markdown
步骤依赖：
  步骤1 角色卡 ← 无依赖（可立即执行）
  步骤2 场景图 ← 无依赖（可并行执行）
  步骤3 分镜图 ← 依赖 步骤1+2（需等待角色和场景锚点）
  步骤4 视频Prompt ← 依赖 步骤1+2+3（需所有资产就绪）
```
