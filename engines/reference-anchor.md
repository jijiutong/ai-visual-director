# 参考锚点引擎

根据目标平台自动分配参考图策略 + 平台硬限制校验。

---

## 一、平台能力矩阵

| 平台 | 时长 | Prompt字数 | 参考图 | 中文 | 多镜头 |
|------|------|-----------|--------|------|--------|
| Seedance | 5-10s | ≤1500字 | 5-6张 | ✅ | ✅ |
| Runway Gen-3 | 5-10s | ≤500字(英文) | 1-3张 | ❌ | ⚠️有限 |
| 可灵 | 5-10s | ≤800字 | 1-3张 | ✅ | ✅ |
| Luma | 5s | ≤300字(英文) | 1-2张 | ❌ | ❌ |
| Sora | ≤60s | ≤500字(英文) | 0-1张 | ❌ | ⚠️ |
| Pika | 3-4s | ≤300字(英文) | 1-2张 | ❌ | ❌ |

---

## 二、自动锚点分配

### Seedance（参考图优先型）
```
5张标准包：场景全景 + 角色卡 + 分镜图 + 关键动作帧 + 尾帧
可选：第二关键帧
```
> @编号和排序以 `state/asset-map.md` 实际映射为准。

### Runway Gen-3（Prompt优先型）
```
3张标准包：首帧 + 关键帧 + 尾帧
角色一致性靠英文 prompt DNA 描述，参考图只给最重要的
```

### 可灵（首帧优先型）
```
3张标准包：首帧(first_frame) + 角色卡 + 关键帧
首帧质量必须最高
```

### Luma / Pika（轻量型）
```
2张最小包：首帧 + 尾帧
英文极简 prompt，不传角色卡
```

---

## 三、一致性策略

| 平台 | 角色一致性 | 场景一致性 |
|------|----------|----------|
| Seedance | 角色卡参考图 | 场景全景参考图 |
| Runway | Prompt DNA 描述 | Prompt 场景描述 |
| 可灵 | 首帧 + Prompt | 首帧 + Prompt |
| Luma/Sora/Pika | Prompt 为主 | Prompt 为主 |

---

## 四、平台校验

```
1. 时长校验：≥平台上限 → 标记"需拆段"
2. 字数校验：>平台上限 → 调用 prompt-compression
3. 参考图校验：>平台上限 → 按优先级裁切（角色卡 > 首帧 > 尾帧 > 关键帧 > 其他）
4. 语言校验：从 `state/platform-config.md` 二（视频平台）和五（图像平台）读取语言支持 → 不匹配则翻译

校验不通过 → auto-repair → 重新校验
```

---

## 五、输出

```markdown

参考图包（[N]张）：
  1. [类型]：[用途] - 用于[镜头编号]
  2. ...

平台校验：
  ✅ 时长：[Ns] — 在限制内
  ✅ 字数：[N字] — 在限制内
  ✅ 参考图：[N张] — 在限制内
  ✅ 语言：[语言] — 平台支持
  ⚠ [问题] → [修复]

平台注意事项：
  - [特殊限制/注意事项]
```

### 同时写入 `state/asset-map.md`（动态 @图映射）

按平台策略生成结构化映射表，供 `video-prompt-assembly` 动态读取：

```markdown
| @编号 | type | name | source | locks |
|-------|------|------|--------|-------|
| @图0 | [scene_reference/character_sheet/end_frame] | [名称] | [来源路径] | [锁定的维度] |
| @图1 | ... | ... | ... | ... |
| @图2 | ... | ... | ... | ... |
```

**映射规则**：
1. 按平台策略决定图片数量和类型（Seedance 5-6张 / Runway 3张 / 可灵 3张 / Luma/Pika 2张）
2. 按资产优先级排序：角色卡 > 首帧/场景图 > 关键帧 > 分镜图 > 道具卡
3. 每张图标注 `type`（类型）、`name`（名称）、`source`（来源路径）、`locks`（锁定维度）
4. 最终 @编号从 0 开始连续编号

> 视频 prompt 生成时，**不硬编码 @图0=场景、@图1=角色**，而是从 `state/asset-map.md` 动态读取映射。

## 联动

← 读取 `state/platform-config.md`（平台限制 + 默认平台 + 拆分→参考图按段分配数量 + 平台语言支持）
← 接收 `asset-plan` 的资产列表
→ 按平台过滤和重新分配
→ **写入 `state/asset-map.md`**（动态生成 @图编号→用途映射表，供 video-prompt-assembly 读取）
→ **更新 `state/variable-registry.md`**（project.word_count 平台校验后更新）
→ 校验发现问题 → `auto-repair`
→ 通过 → `motion-physics`
→ **平台限制从此文件读取，不硬编码**。修改 `state/platform-config.md` 即全局生效。
