# 故事摄入引擎

把用户故事自动拆成可生产字段。这是整个自动决策链的第一环。

---

## 提取字段

| 字段 | 优先级 | 缺省行为 |
|------|--------|---------|
| 类型/Genre | 必须 | 按冲突强度推断 |
| 主角/Protagonist | 必须 | 给起电影感名字 |
| 目标/Goal | 必须 | "活下来" / "战胜对手" |
| 冲突/Conflict | 必须 | 谁 vs 谁 / 什么 |
| 主场景/Primary Scene | 必须 | 提取第一个明确地点 |
| 情绪底色/Emotion | 重要 | 按冲突类型推断 |
| 时长/Duration | 可选 | 由 downstream shot-budget 根据内容判断，不从 story-intake 设默认 |
| 视频平台/Target Platform | 可选 | 从 `api-config.template.env` 读取 `VIDEO_PLATFORM_DEFAULT` |
| 画幅/Aspect Ratio | 可选 | 从 `api-config.template.env` 读取 `DEFAULT_ASPECT_RATIO` |
| 角色数量 | 必须 | 提取所有命名角色 + 功能角色 |
| 场景数量 | 必须 | 提取所有明确地点 |
| 时间跨度 | 可选 | 从故事推断（瞬间/几小时/一天/多天） |
| 动作复杂度 | 可选 | 低（对话）/ 中（有动作）/ 高（多场打斗） |

---

## 决策逻辑

### 步骤1：解析故事类型

| 关键词 | 映射类型 | 默认节奏 |
|--------|---------|---------|
| 剑/刀/复仇/恩怨 | 武侠 | P1 快切 |
| 机甲/太空/AI/赛博 | 科幻 | P1 快切 |
| 修仙/灵气/仙/神魔 | 玄幻 | P2 标准→P1高潮 |
| 恋/爱/暗恋/分手 | 爱情 | P3 慢节奏 |
| 鬼/恐怖/诅咒/死 | 恐怖 | P4 渐进 |
| 笑/搞笑/乌龙/奇葩 | 喜剧 | P5 喜剧 |
| 校园/青春/毕业 | 青春 | P2 标准 |
| 战争/军队/战役 | 战争 | P1 快切 |
| 悬疑/侦探/推理 | 悬疑 | P4 渐进 |
| 末日/废土/丧尸 | 末日 | P2 标准 |
| （无关键词匹配） | 剧情 | P2 标准 |

---

### 步骤2：识别角色

```
规则：
- 每个被命名的角色 → 提取姓名/身份/关键词描述
- 未命名但反复出现的角色 → 按功能命名（"剑客""少女""老人"）
- 单角色故事：主角1人
- 双角色故事：主角+对手/搭档
- 多角色故事（≥3人）：标记"建议拆段"（最终由 shot-budget 判断，≤4人仍可单次生成）
```

---

### 步骤3：判断视频时长（初步估算）

以下为故事复杂度→时长估算参考。最终时长由 `shot-budget` 结合 api-config.template.env 平台上限决定。

| 故事复杂度 | 估算时长 | 理由 |
|-----------|---------|------|
| 1个场景 + 1个动作 | ~10s | 简单瞬间 |
| 1个场景 + 2-3个节拍 | ~15s | 默认短篇 |
| 2个场景 + 情绪弧线 | ~30s | 需空间转换 |
| 3+场景 + 完整起承转合 | ~60s | 完整叙事 |
| 长篇（≥500字） | 拆段 | 标记"不可单次生成" |

> 以上为估算参考，非硬默认。最终时长/镜数/是否拆段由 `engines/shot-budget.md` 根据平台上限 + pacing 表计算。

---

### 步骤4：输出生产卡片

```markdown
【故事摄入结果】

片名：[提取/推断]
类型：[类型] → 节奏[P编号]
时长：[Ns] / 画幅：[比例] / 平台：[平台名]

主角：[姓名/身份/一句话描述]
对手：[姓名/身份/一句话描述]
配角：[列表]

场景：[主场景] → [辅助场景]
时间跨度：[推断] / 动作复杂度：[低/中/高]

冲突：[一句话核心冲突]

视频判断：[一次生成] / [拆N段]
```
