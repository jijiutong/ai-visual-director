# 一致性触达决策引擎

在 `/create` 主链中位于 `asset-plan` 之后、`reference-anchor` 之前。读取上游 state/ 数据，对分镜图密度、场景一致性方法、角色一致性方法做出智能决策，输出人类可读的保障建议。**不阻塞主链**，仅建议。

---

## 职责边界

| 引擎 | 职责 |
|------|------|
| `asset-plan` | 规划「需要哪些资产类型」（角色卡/场景图/分镜图/关键帧） |
| **`consistency-trigger`（本引擎）** | 判断「这些资产用哪种一致性方法生成」+「分镜图拆几张」 |
| `reference-anchor` | 分配 @图编号、平台校验 |

> ⚠ 本引擎不做一致性评估（那是 consistency-engine 的活），也不分配 @图（那是 reference-anchor 的活）。只做**方法选择决策**。

---

## 一、读取上游状态

| 读取参数 | 来源 | 用途 |
|---------|------|------|
| 总镜数 | `state/shot-state.md` → shots 数组长度 | 分镜密度 |
| 各镜景别分布 | `state/shot-state.md` → shots[].shot_size | 场景复杂度（仰/平/俯角度分布） |
| 视频类型/genre | `state/variable-registry.md` → story.genre | 动作场景加严 |
| 场景数 | `state/variable-registry.md` → scene.* 条目数 | 场景一致性等级 |
| 场景空间复杂度 | `state/variable-registry.md` → scene.primary.fixed_elements 数量 | 场景方法选择 |
| 角色数 | `state/variable-registry.md` → characters.* 条目数 | 角色一致性等级 |
| 角色出镜分布 | `state/project-graph.md` → 正向索引 character→shots | 主角/配角判断 |
| 服装复杂度 | `state/variable-registry.md` → characters.*.costume / weapon / accessories | 角色方法升级 |
| 预算级别 | `asset-plan` 输出 → 预算级别（极简/标准/精细/电影级） | 整体降级/升级 |

## 二、分镜图密度决策

> 目标：根据镜数和复杂度，决定分镜图拆成几张。避免「15s/7镜挤一张图看不清」。

### 决策表

| 总镜数 | 视频类型 | 推荐方案 | 出图量 | 每图帧数 | 适用场景 |
|--------|---------|---------|--------|---------|---------|
| ≤3镜 | 任意 | 1图全帧 | 1张 | 3帧/图 | 快速验证、简单情绪戏 |
| 4-7镜 | 剧情/对话/情绪 | 标准2图 | 2张 | 3-4帧/图 | 大多数商业片、短剧 |
| 4-7镜 | 动作/战斗/追车 | 精细3图 | 3张 | 2-3帧/图 | 动作细节需要更大画面 |
| 8-10镜 | 任意 | 精细拆分 | 3-4张 | 2-3帧/图 | 长片段、多阶段 |
| >10镜 | 任意 | 逐帧模式 | N/2 或 N张 | 1-2帧/图 | 史诗级、关键战役 |

### 特殊规则

- **拆段场景**：每段独立决策。段1（0-15s）7镜 → 3图；段2（15-30s）7镜 → 3图。不是全局7图
- **预算降级**：如果 budget_level = 极简，强制降至 1-2图 忽略镜数
- **高精升级**：如果 budget_level = 电影级，>4镜自动升级到下一档

### 输出样例

```
📐 分镜图密度：
  当前 7镜，动作类型 → 建议 3张图（每张含2-3帧）
  拆分：图1(SH01-SH03 踏入→触碰) / 图2(SH04-SH05 共鸣→降光) / 图3(SH06-SH07 举剑→觉醒)
  原因：7镜单张细节不够，动作场景每帧需更大画幅才能看清
  预估费用：~$0.24（3张 × ~$0.08/张）
```

## 三、场景一致性触达决策

> 目标：根据场景数、拍摄角度分布、空间复杂度，从 7 种方法中推荐最优方案。

### 场景复杂度评估

从 `state/shot-state.md` 中提取各镜 shot_size 分布：

| 信号 | 含义 | 风险 |
|------|------|------|
| 仅有 FS/LS 远景 | 正面为主，角度单一 | 低 |
| 含 MCU/CU 近景 | 接近拍摄对象 | 中 |
| 含 ECU 极近特写 | 细节放大 | 中 |
| 含 仰拍(LA)+俯拍(HA)+平拍 | 多角度 | 高 |
| 场景 fixed_elements ≥ 5个 | 空间元素多 | 高 |
| 多场景切换（scene 条目 ≥ 2） | 跨场景 | 严重 |

### 决策表

| 场景数 | 角度分布 | 空间复杂度 | 推荐方法 | 出图量 | 费用 |
|--------|---------|-----------|---------|--------|------|
| 1场景 | 正面为主 | 简单（≤3个固定元素） | **全能参考图**（方法4） | 1张 | ~$0.08 |
| 1场景 | 含仰/俯 2-3角度 | 中等（4-6个固定元素） | **4宫格**（方法2） | 1张 | ~$0.08 |
| 1场景 | 仰+平+俯 全角度 | 复杂（≥7个固定元素） | **九宫格**（方法3） | 1张 | ~$0.08 |
| 1场景 | 全角度 + 需要任一方向取景 | 大型（神殿/战场/城市） | **720°全景图**（方法5） | 6张 | ~$0.48 |
| ≥2场景 | — | — | **每场景独立参考图** | N张 | ~$0.08×N |
| 任意 | 优先视频截图法 | 需要最高锁定强度 | **环绕视频截图法**（方法6）| 1视频+N图 | ~$1.0 |

### 降级规则

| 预算级别 | 降级行为 |
|---------|---------|
| 极简 | 720°全景 → 全能参考图；多场景 → 仅主场景1张 |
| 标准 | 不做降级 |
| 精细/电影级 | 自动升级一档（九宫格→720°全景） |

### 知识库交叉引用（必须执行）

**决策完成后，必须到 `rules/scene-consistency.md` 中搜索推荐方法，拉取完整定义**。不能只用简化名。

```
决策输出：推荐「九宫格」方法3
  ↓
搜索 rules/scene-consistency.md → 三、九宫格（9-Grid）
  ↓
拉取详情：
  · 完整名称：九宫格（9-Grid）
  · 出图量：1张
  · 费用：~$0.04-0.08
  · 耗时：~40s
  · 锁定强度：⭐⭐⭐
  · 适用：复杂场景（城镇/战场/宫殿/森林/洞穴）
  · 优点：覆盖全面，复杂场景不遗漏
  · 缺点：生成成本较高
  · 触发命令：「九宫格」/「9宫格」
  · 备选降级：4宫格（方法2，~$0.04-0.08）
  · 备选升级：720°全景（方法5，~$0.24-0.48）
```

> ⚠ 知识库里 7 种方法的费用/耗时/强度/优缺点各不相同，不可用决策表里简化版替代。必须去知识库搜。搜索不到的"方法"说明不存在，不能编造。

### 输出样例

```
🏛 场景一致性：
  当前「碎穹神殿废墟」1场景，多角度拍摄（仰/平/俯全含），7个固定元素
  → 建议「九宫格场景锁定」（方法3，1张图9视角）
  原因：神像石手/苍穹裂缝/悬浮碎片/石柱群/地面裂缝/苔藓/灵光粒子 元素多 + 全角度拍摄
  
  【知识库参数】（来自 rules/scene-consistency.md）：
  · 方法3 九宫格（9-Grid）：1张 | ~$0.04-0.08 | ⭐⭐⭐ | ~40s
  · 优点：覆盖全面，复杂场景不遗漏 | 缺点：生成成本较高
  · 适用：复杂场景（城镇/战场/宫殿/森林/洞穴）
  · 备选降级：4宫格（方法2，~$0.04-0.08）— 牺牲2个角度
  · 备选升级：720°全景（方法5，~$0.24-0.48）— 覆盖所有拍摄方向
```

## 四、角色一致性触达决策

> 目标：根据角色重要度、出镜率、服装/武器复杂度，从 8 种方法中推荐最优方案。

### 角色重要度评估

从 `state/project-graph.md` 计算每个角色的出镜占比：

```
出镜占比 = character→shots 数量 / 总镜数
重要度：
  主角：占比 ≥ 50% 或 出镜 ≥ 4镜
  重要配角：占比 30%-49% 或 出镜 2-3镜
  配角：占比 < 30% 且 出镜 ≤ 1镜
```

### 决策表

| 重要度 | 出镜数 | 服装/武器复杂度 | 推荐方法 | 出图量 | 费用 |
|--------|--------|---------------|---------|--------|------|
| 配角 | ≤1镜 | 简单 | **角色卡**（方法1）+ DNA文字（方法8） | 1张 | ~$0.08 |
| 配角 | 2-3镜 | 简单 | **角色卡**（方法1）+ DNA文字（方法8） | 1张 | ~$0.08 |
| 主角 | >2镜 | 中等（单层服装） | **角色卡**（方法1）+ **面部5角度**（方法3） | 2张 | ~$0.16 |
| 主角 | >4镜 | 复杂（多层服装/铠甲/特殊材质） | + **服装武器细节卡**（方法5） | 3张 | ~$0.24 |
| 主角 | >6镜 | 复杂 + 多套服装/变身 | + **面部12表情**（方法4） | 4张 | ~$0.32 |
| 主角 | >6镜 + 电影级预算 | 复杂 + 多变 | + **面部12表情** + **服装层次爆炸图** | 5张 | ~$0.40 |

### 特殊规则

- **多角色项目**：每个主角独立评估，配角共享降级规则
- **武器道具角色**：如果角色有 `weapon` 字段且非空 → 自动 +服装武器细节卡
- **变身/战损**：如果 story 描述含「变身/黑化/战损/进化」→ 自动 +面部12表情
- **IP-Adapter 提示**：如果预算 = 电影级 → 追加提示「ComfyUI IP-Adapter（零成本无限生成，最强一致性）」

### 知识库交叉引用（必须执行）

**决策完成后，必须到 `rules/character-consistency.md` 中搜索每个推荐方法，拉取完整定义**。

```
决策输出：凌苍（主角）→ 推荐方法1+方法3+方法5（3张）
  ↓
搜索 rules/character-consistency.md → 一、角色设定卡 / 三、面部多角度 / 五、服装武器细节卡
  ↓
拉取详情（每人每方法）：
  · 方法1 角色设定卡（6模块）：1张 | ~$0.04-0.08 | ⭐⭐⭐ | ~40s
    → 适用：全部平台 | 触发：「生成角色卡」
  · 方法3 面部多角度（5角度）：1张 | ~$0.04-0.08 | ⭐⭐⭐⭐ | ~40s
    → 适用：全部平台 | 触发：「面部多角度」/「5角度」
  · 方法5 服装武器细节卡：1张 | ~$0.04-0.08 | ⭐⭐⭐ | ~30s
    → 适用：全部平台 | 触发：「服装武器细节卡」
```

> ⚠ 知识库里 8 种方法的费用/耗时/强度/平台兼容各不相同。决策表只做触发判断（用不用这个方法），知识库给完整参数（这个方法多少钱/多快/多强）。必须两方结合输出。

### 输出样例

```

  凌苍（主角）：7/7镜全出镜，复杂服装（破旧黑袍+锁链），特殊武器（碎穹剑）
```

## 五、组合输出格式

在 `/create` 最终 `render-package` 输出前追加：

```markdown
【一致性保障建议】

📐 分镜图密度：
  [方案描述]
  [拆分说明]
  [费用]

🏛 场景一致性：（搜索自 rules/scene-consistency.md）
  [推荐方法 + 编号]
  [知识库参数：出图量 | 费用 | 耗时 | 锁定强度 | 适用场景]
  [优点/缺点]
  [原因]
  [备选降级方案]

👤 角色一致性：（搜索自 rules/character-consistency.md）
  [每个角色独立一行]
  [推荐方法列表]
  [知识库参数 per 方法：出图量 | 费用 | 耗时 | 锁定强度 | 平台兼容]
  [原因]

──────────────────────────
总一致性保障：[N项] / [N张额外图] / ~$[总费用]

回复「全部出」→ 一键生成所有保障图
回复「精简」  → 降至最低方案（全能参考图 + 角色卡 + DNA文字）
回复「跳过」  → 忽略建议，直接输出
```

> ⚠ 本引擎输出是**建议**，不是命令。用户可选择跳过、升级或降级。主链继续执行 `reference-anchor`，不等待用户回复。

## 六、不阻塞主链原则

```
consistency-trigger 输出建议
  ↓
reference-anchor（继续执行，不等用户回复）
  ↓
motion-physics → project-graph → 子路由 → video-prompt-assembly → ...
  ↓
render-package（最终输出时附带建议区块）
```

> 建议作为 `render-package` 的前置区块，用户在拿到完整 prompt 的同时看到一致性建议。

## 联动

← 读取 `state/shot-state.md`（镜数 + 景别分布）
← 读取 `state/variable-registry.md`（类型 + 场景 + 角色 + 服装）
← 读取 `state/project-graph.md`（角色→镜头分布）
← 接收 `asset-plan` 的资产规划结果（预算级别 + 已规划的资产类型）
→ **搜索 `rules/scene-consistency.md`** — 拉取推荐方法的完整参数（出图量/费用/耗时/强度/优缺点/触发命令），不可用简化版替代
→ **搜索 `rules/character-consistency.md`** — 同上，每个推荐方法拉取完整参数（出图量/费用/耗时/强度/平台兼容/触发命令）
→ 输出合并后的保障建议（决策逻辑 + 知识库参数）到 `render-package`
→ 不阻塞主链（建议仅附加，不改变执行流）
