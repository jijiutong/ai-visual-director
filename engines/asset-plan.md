# 资产规划引擎

根据视频类型和复杂度，自动决定需要生成哪些参考资产。含最低资产硬性要求。

---

## 一、按视频类型的最低资产

### 类型匹配（从 story-intake 的类型 → 资产类型）

| story-intake 类型 | → 资产类型 | 条件 |
|------------------|-----------|------|
| 武侠/战争 | 战斗/动作 | 有打斗描述；否则 → 双人对峙 |
| 科幻 | 科幻/机甲 | 有机甲/装备描述；否则按人数匹配 |
| 玄幻 | 大场景神话 | 有神魔/巨物；否则按人数匹配 |
| 爱情 | 双人对峙/对话 | 默认；单角色 → 单人情绪戏 |
| 恐怖/悬疑 | 单人情绪戏 | 默认；多角色 → 双人对峙 |
| 喜剧/青春 | 单人情绪戏 | 根据场景调整 |
| 末日 | 大场景神话 | 有宏大场景；否则按人数匹配 |
| 剧情（默认） | 按实际角色+场景内容匹配 | 单角色→单人 / 双角色→双人 / 多角色→群像 / 有战斗→战斗 |

### 单人情绪戏
```
☑ 角色卡 ×1（模式A 6模块全量版）
☑ 首帧 ×1 + 尾帧 ×1
不满足 → 禁止输出视频 prompt
```

### 双人对峙/对话
```
☑ 角色卡 ×2（双方各自）
☑ 场景空间关系图 ×1（LS24 双人关系板）
☑ 首帧 ×1
不满足 → 禁止输出视频 prompt
```

### 战斗/动作
```
☑ 角色卡 ×1（主角）
☑ 武器/道具卡 ×1（如有特殊武器）
☑ 动作关键帧 ×3（起势/高潮/收势）
☑ 首帧 ×1 + 尾帧 ×1
不满足 → 禁止输出视频 prompt
```

### 大场景神话/奇幻
```
☑ 场景全景 ×1（LS8 场景全能参考板）
☑ 角色卡 ×1（比例衬托）
☑ 全案分镜图 ×1（LS6 横幅时间轴）
不满足 → 禁止输出视频 prompt
```

### 多角色群像（≥3人）
```
☑ 角色卡 ×N（每个主要角色）
☑ 站位俯视图 ×1（LS27 场景蓝图俯视板）
☑ 全案分镜图 ×1（LS15 导演作战室）
不满足 → 禁止输出视频 prompt
```

### 科幻/机甲
```
☑ 角色卡 ×1（LS13 科幻神使系统卡）
☑ 机甲/装备拆解卡 ×1（LS22 服装层次爆炸图）
☑ 场景图 ×1（科幻环境）
不满足 → 禁止输出视频 prompt
```

### 神兽/生物
```
☑ 生物角色卡 ×1 + 比例尺 ×1 + 场景图 ×1
不满足 → 禁止输出视频 prompt
```

### 追车/追逐
```
☑ 场景路线图 ×1 + 角色卡 ×1 + 关键帧 ×2
不满足 → 禁止输出视频 prompt
```

## 二、资产包等级（最低强制）

> **强制规则来源**：`rules/asset-minimums.md`。本引擎规划资产时必须对照该规则，触发强制条件时不可降级。

| 级别 | 对应资产包 | 触发条件 |
|------|----------|---------|
| 极简（快速包） | 角色卡 + 场景卡 + 分镜图 | 仅 ≤3镜/单角色/单场景/简单服装/无武器 全部满足 |
| 标准（标准包） | 快速包 + 首帧 | 默认，大多数情况 |
| 精细（正式包） | 标准包 + 面部多角度 + 服饰细节 + 场景九宫格 + 尾帧 | 主角>4镜 或 复杂服装 或 多角度场景 |
| 电影级（长篇包） | 正式包 + continuity-snapshot + project-graph | 多段/多章/系列 |

> 强制升级条件详见 `rules/asset-minimums.md` 第二节。不满足最低资产时，`consistency-trigger` 阻断主链。

## 三、生成顺序

```
必须先出（锁身份和空间）：
  1. 角色卡（锁谁）
  2. 场景全景（锁在哪）

然后出（锁动作和关系）：
  3. 关键动作帧（锁怎么动）
  4. 空间关系图（锁谁和谁什么关系）

最后出（锁起点终点）：
  5. 首帧（镜头1）
  6. 尾帧（最后镜头）
```

## 四、资产按段分配

当内容被拆为多段时，资产按以下策略分配。非拆段场景（单段）则全部产出。

| 资产类型 | 分配策略 |
|---------|---------|
| 场景图 | 跨段复用（同一场景）或按段独立（不同场景） |
| 角色卡 | 跨段复用（角色 DNA 不变） |
| **全案分镜图** | **每段 1 张**（7镜以内一张图清晰可读，>7镜拆多张） |
| 高潮关键帧 | 每段 1 张（每段有自己的高潮镜） |
| 首帧 | 段1首帧 |
| 尾帧 | 最后一段尾帧 |
| 段间接力 | 段N尾帧 = 段N+1首帧（自动锚定） |

> **禁止**：60s 视频只出 1 张全案图塞 15 镜——看不清、视频工具也用不了。

|------|---------|
| 角色卡 | 所有含该角色的镜头 |

## 六、资产质量检查

| 检查项 | 要求 |
|--------|------|
| 角色卡 | 五官/发型/服装/体型不可跨镜变化 |
| 场景图 | 主光源方向明确、空间关系清晰 |
| 首尾帧 | 同一角色、同一场景、同一色调 |
| 关键动作帧 | 运动方向明确、动作可执行 |
| 空间关系图 | 角色站位、间距、视线方向清楚 |

## 七、资产用途分类（强制分流）

> 所有资产必须标记用途。不同用途的图进入不同流程，禁止混用。

| 用途类型 | 标识 | 定义 | 进入流程 | 典型资产 |
|---------|------|------|---------|---------|
| **display_asset** | 📋 展示 | 给人看的设定图，允许文字/边框/标注/高密度信息 | storyboard/character/scene 展示输出，不进视频参考 | 全案板、角色圣经、设定集、分镜表 |
| **video_asset** | 🎬 视频 | 给 AI 视频模型的参考图，必须主体干净/少文字/少UI/无边框 | 进入视频 prompt @图引用列表 | clean 角色卡、clean 场景锚点、首尾帧 |
| **consistency_asset** | 🔒 一致性 | 专门锁角色/场景一致性的结构化参考图 | 进入 consistency-engine 评估 + 可选进入视频参考 | 九宫格场景锁定、面部5角度、三视图 |
| **marketing_asset** | 📢 营销 | 封面/海报/宣传物料，高冲击但**不进入视频参考** | 仅 poster/封面输出 | 电影海报、B站封面、宣传横幅 |

### 资产用途归类表

| 资产 | 默认用途 | 可否派生 video_asset | 说明 |
|------|---------|-------------------|------|
| 全案板（full-board） | display_asset | ✅ 拆出 central hero frame | 高密度排版，AI 视频工具无法解析 |
| 角色板（黑金/科幻/展示向） | display_asset | ✅ 生成 clean character sheet | 文字/箭头/HUD 污染视频模型 |
| 角色卡（模式A 6模块全量版） | video_asset | — | 已符合视频锚点标准 |
| 三视图 | consistency_asset | — | 正交视图锁角色体型 |
| 面部多角度 | consistency_asset | — | 锁面部结构 |
| 十二表情板 | consistency_asset | — | 锁表情范围 |
| 场景图（全能参考/LS8） | display_asset | ✅ 生成 no-text scene anchor | 含材质/灯光/声音标注，非纯空间 |
| 场景九宫格（LS9） | consistency_asset | ✅ 已是空间锁定 | 9 格空间结构完整 |
| 分镜图（full-board/quick-board） | display_asset | ✅ 拆出无文字 keyframes | 含技术参数/台词/标注 |
| 海报/封面 | marketing_asset | ❌ 不进视频 | 高冲击排版，AI 视频工具无法参考 |
| 情绪板 | display_asset | ❌ | 色彩/材质参考，非空间锚点 |
| 关键帧序列 | display_asset | ✅ 拆出 clean keyframes | 可清洁后用作视频锚点 |
| 首帧/尾帧 | video_asset | — | 直接用于视频起终点锚定 |
| 世界观板 | display_asset | ❌ | 信息密度极高 |
| HUD 科技卡 | display_asset | ❌ | 大量伪 UI，污染视频模型 |

### 自动提示规则

当用户产出全部为 display_asset / marketing_asset 时，系统自动提示：

```markdown
⚠ 当前资产均为展示用途，无法直接喂给视频模型。

已有：
  📋 全案板（display）— 好看但不能用作视频参考
  📋 角色板（display）— 文字/箭头会污染视频模型
  📋 分镜图（display）— 技术标注会干扰 AI 理解

建议生成 video_asset：
  🎬 clean 角色卡（模式A 6模块全量版）
  🎬 clean 场景锚点（九宫格空间锁定 LS9）
  🎬 视频分镜图（合并帧 LS7，无文字无标注）

回复「生成视频锚点」→ 一键生成全部 video_asset
回复「跳过」→ 继续用当前资产（视频一致性不保证）
```

## 资产变更传播

当角色卡或场景图重新生成（immutable_features / fixed_elements 变更）时，**必须**通过 project-graph 向下游传播：

```
资产变更（variable-registry 中的 immutable_features 或 fixed_elements 更新）
  ↓
1. 调用 engines/incremental-update.md：
   ├─ entity_type = character（角色卡变更）或 scene（场景图变更）
   ├─ 查询 state/project-graph.md → affected_shots / affected_assets
   └─ 获得精确的影响范围（不再手动扫描全部 shot-state）
2. 更新受影响镜头：
   ├─ 角色变更 → 更新各镜 action 中的外观描述
   ├─ 场景变更 → 更新各镜 lighting/color 中的空间约束
   └─ 同步标记 asset-map 中需要重新生成的 @图
3. 触发 consistency-engine（增量模式）：
   └─ 只评估受影响的 RM 维度
4. 输出变更影响报告
```

### 示例

```
重新生成角色卡 "墨渊" → immutable_features 更新：
  旧：黑衣/长发/暗金护腕
  新：黑衣/长发/暗金护腕/左脸疤痕/「断念」剑身裂纹

project-graph 查询：
  affected_shots("character", "C1") → [SH01, SH03, SH05, SH07]
  affected_assets("character", "C1") → [@图1]

传播：
  → SH1 action 追加 "left cheek scar visible"
  → SH3 action "剑身细密裂纹泛青光"
  → SH5 action "疤痕在闪电下格外醒目"
  → SH7 尾帧 "裂纹沿剑身蔓延至剑尖"
  → @图1 标记「⚠ 待重新生成」

受影响: 4镜（SH1/3/5/7）| 跳过: 3镜（SH2/4/6）| 重评: 仅 Character RM
```

## 联动

← 读取 `api-config.template.env`（平台默认 + 拆分规则）
← §四 资产按段分配（段数 → 全案图：每段1张。禁止60s只出1张全案图）
← 接收 `video-director` 的参考图决策 + `story-intake` 的视频类型
→ 输出资产列表（含用途分类 📋/🎬/🔒/📢）给模板引擎
→ 用途分类写入 `state/asset-map.md`（asset_purpose 字段）
→ 不满足最低资产 → `auto-repair` 触发修复
→ 满足 → 放行到 `reference-anchor`
→ **写入 `state/variable-registry.md`**（style.layout_*, characters.*.immutable_features, scene.primary.fixed_elements）
→ **资产变更时触发传播**：调用 `engines/incremental-update.md` → 查询 `state/project-graph.md` → 更新关联镜头 → 增量重评 consistency-engine
