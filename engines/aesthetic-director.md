# 审美导演

把用户的口语审美描述转译为结构化 10 字段。口语不能直接进入 prompt。本引擎作为 `/character` 子链和 `prompt-qc` 维度5 的翻译层。

---

## 一、核心原则

```
口语审美 → 本引擎转译 → 10字段结构化描述 → prompt-qc 检查 → 写入 prompt

禁止：口语直接写入 prompt
禁止：跳过转译直接使用用户原词
```

---

## 二、10 字段转译框架

每个角色人像必须覆盖以下 10 字段：

| # | 字段 | 说明 | 示例 |
|---|------|------|------|
| 1 | **脸型** | 鹅蛋脸/V脸/方圆脸/心形脸/圆脸/方脸 | 小鹅蛋脸，下颌线柔和 |
| 2 | **五官-眼** | 眼型+双眼皮+眼神特征 | 杏眼，眼尾微挑，双眼皮自然宽度，眼睛清透有卧蚕 |
| 3 | **五官-鼻** | 鼻梁高度+鼻头形状 | 高鼻梁，鼻头小巧自然 |
| 4 | **五官-唇** | 唇形+厚度+唇色 | M唇，饱满自然，裸粉色 |
| 5 | **妆容** | 底妆+眼妆+腮红+修容 | 轻透底妆，精致眼妆不过浓，自然腮红 |
| 6 | **发型** | 长度+卷直+颜色+质感 | 微卷长发，深棕色，自然蓬松 |
| 7 | **肤质** | 质感描述（避免硅胶感） | 自然肤质，真实皮肤纹理，微光泽 |
| 8 | **镜头** | 焦段+构图 | 85mm 人像镜头，浅景深 |
| 9 | **光线** | 光位+光质 | 柔和环形补光，正面略偏上方 |
| 10 | **气质** | 角色气质+表情方向 | 亲近明亮，自然微笑，非摆拍感 |

## 四、类型与审美冲突检测

如果用户口语审美与角色类型冲突，先提示再转译：

| 冲突 | 处理 |
|------|------|
| 武侠女侠 + 网红美女 | 提示冲突："网红美女"妆感与武侠战斗场景不匹配。建议：自然裸妆、有力量感。用户确认后转译 |
| 末日废土 + 甜妹 | 提示冲突：末日环境与甜美感矛盾。建议：坚毅自然感 |
| 古风仙气 + 高级脸 | 不冲突，可融合（古风骨相+高级脸留白） |
| 科幻特工 + 网红美女 | 轻微冲突，提示：建议选一个方向为主 |

## 五、与 prompt-qc 的协作

```
用户输入口语审美
  ↓
aesthetic-director（本引擎）→ 转译 10 字段
  ↓
写入 role prompt
  ↓
prompt-qc 维度5（审美合同）→ 检查：
  □ 年龄明确？
  □ 脸型具体？
  □ 五官各字段完整？
  □ 无口语残留？
  □ 负面词齐全？
  ↓
不通过 → auto-repair 补充
通过 → 放行
```

---

## 六、转译规则

1. **不允许跳过**：有口语审美输入就必须转译，不能"用户没说清楚"就放行
2. **不允许简化**：10 字段缺一不可。缺失字段从角色类型推荐值补全
3. **负面词必加**：每次转译必须追加对应类型的防丑负面词
4. **冲突提示**：检测到类型与审美矛盾时先提示，不静默覆盖
5. **用户覆盖优先**：用户明确写出的具体字段（如"丹凤眼"）不被转译覆盖
