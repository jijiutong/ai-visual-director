# Prompt 压缩引擎

当 prompt 超平台字数限制时，自动压缩。保留关键信息，删除冗余描述。

---

## 一、压缩策略

| 优先级 | 操作 | 说明 |
|--------|------|------|
| P0 保留 | 角色DNA + 场景锚点 + 动作方向 + 光线方向 | 绝对不能删 |
| P1 保留 | 镜头顺序 + 运动类型 + 情绪关键词 | 结构必须保留 |
| P2 压缩 | 修辞形容词 → 精准关键词 | "缓缓地非常优雅地"→"缓步" |
| P3 压缩 | 合并重复禁止词 | "不要多余的不要额外的"→"不要多余元素" |
| P4 删除 | 氛围铺陈 | "空气中弥漫着..." → 删 |
| P5 删除 | 多余的镜头描述 | 每个镜头保留1句核心动作 |
| P6 删除 | 平台不支持的参数 | 如 Runway 删 --cref |

---

## 二、字符预算

**读取 `api-config.template.env`（视频平台硬限制）**，获取目标平台的 prompt 字数上限：

```
platform = state/variable-registry.project.target_platform（或用户指定平台）
max_chars = api-config.template.env 该平台的「最大 prompt 字数」
安全线 = max_chars × 0.9
```

> 平台字数限制不硬编码在此文件中，全部从 `api-config.template.env` 读取。修改 api-config.template.env 即全局生效。

超安全线 → 立即压缩到安全线以下。

---

## 三、压缩操作

### 中文 → 中文压缩

```
原文：在昏暗的雨夜里，一道闪电划破天际，照亮了古老的剑冢，雨水顺着残破的石碑缓缓流下
压缩：雨夜闪电照亮剑冢，雨水顺石碑流下

原文：主角慢慢地、非常优雅地、带着一种不可言说的悲伤转过身去
压缩：主角缓缓转身，神色悲伤
```

### 中文 → 英文压缩

```
1. 先压缩中文到平台英文限制对应长度（中文1字 ≈ 英文2-3词 → 上限÷2.5）
2. 翻译为英文
3. 英文再微调到安全线以下
```

### 常用压缩规则

| 原文模式 | 压缩为 |
|---------|--------|
| "在...的...中" 句式 | 删"在"和"中"，留核心词 |
| "非常/极其/特别/十分" | 删 |
| "缓缓地/迅速地/猛烈地" | 删"地" → "缓步/疾冲/猛击" |
| 重复的禁止词 | 合并为1句 |
| "画面中/镜头里" | 删（AI知道在描述画面） |
| 长串形容词 | 保留最核心2个 |

---

## 四、压缩流程

```
1. 计算当前字数
2. 超安全线？→ 进入压缩
3. P6→P2 逐级删除，每级后重新计算
4. 达到安全线以下 → 停止
5. 超上限但压缩不掉？→ 标记"需拆段"
```

## 联动

← **读取 `api-config.template.env`** → 获取目标平台字数上限（不硬编码）

← 被 `auto-repair` 策略4触发
→ 压缩后返回调用方
→ 仍超标 → 标记"需拆段"
