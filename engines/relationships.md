# 角色关系引擎

根据角色互动张力自动匹配关系类型。数据表见 `knowledge/relationships.md`。

---

## 决策逻辑

### 步骤1：识别关系类型

从 `state/variable-registry.md` 读取角色互动信息，匹配关系编号：

| 角色互动特征 | 推荐关系编号 |
|-------------|-------------|
| 对抗→心动、相爱相杀 | R1 宿敌变恋人 |
| 试探、闪躲、暗恋 | R2 双向暗恋 |
| 师傅/徒弟、教导 | R3 师徒 |
| 身高差、年龄差 | R4 年上/年下 |
| 契约婚姻、先冷后热 | R5 先婚后爱 |
| 互怼、欢喜冤家 | R6 死对头/冤家 |
| 黑暗→光、治愈 | R7 救赎/治愈 |
| 替身、白月光 | R8 替身/白月光 |
| 控制、禁锢 | R9 强制/囚禁 |
| 战友、搭档、信任 | R10 战友/搭档 |
| 重逢、破镜重圆 | R11 重逢/破镜重圆 |
| 阵营对立、身份冲突 | R12 身份对立 |
| 养成、守护 | R13 养成/守护 |
| 暧昧、拉扯 | R14 暧昧拉扯 |
| 一见钟情 | R15 一见钟情 |
| 血缘、禁忌 | R16 血缘禁忌 |
| 间谍、双面 | R17 双面间谍 |
| 失忆、重逢 | R18 失忆重逢 |
| 克隆体、复制人 | R19 克隆体冲突 |
| AI、机器人 | R20 AI与人类 |
| 妖、异类 | R21 跨物种之恋 |
| 时间线、穿越 | R22 时间线交错 |
| 人格分裂、镜像 | R23 镜像人格 |
| 情敌、同盟 | R24 情敌同盟 |
| 恩人、背叛 | R25 恩人变仇人 |
| 假戏、演戏 | R26 假戏真做 |
| 前世、轮回 | R27 前世今生 |
| 上下级、权力 | R28 上下级/权力差 |
| 竞争、对手 | R29 竞争者→盟友 |

| 契约、主仆 | R31 契约主仆 |
| 成长、蜕变 | R32 共同成长 |

> 完整关系定义见 `knowledge/relationships.md`

---

### 步骤2：获取关系参数

根据关系编号，从 `knowledge/relationships.md` 获取：
- 构图建议
- 镜头语言
- 灯光方案
- 氛围关键词
- Prompt 追加模板

---

### 步骤3：风格匹配加成

每种关系有最佳风格匹配，自动追加到风格推荐中：

| 关系类型 | 风格加成 |
|---------|---------|
| R1 宿敌变恋人 | VS1 黑金动作 / VS7 东方玄幻 |
| R2 双向暗恋 | VS37 都市情绪 / VS38 青春校园 |
| R3 师徒 | VS7 东方玄幻 / VS24 吉卜力 |
| R7 救赎/治愈 | VS37 都市情绪 / VS24 吉卜力 |
| R9 强制/囚禁 | VS15 暗黑哥特 / VS39 悬疑惊悚 |
| R10 战友/搭档 | VS1 黑金动作 / VS2 科幻机甲 |

> 完整匹配表见 `knowledge/relationships.md` 速查表

---

### 步骤4：输出关系加成

整合关系编号 + 关系参数，输出关系加成 Prompt。

---

## Prompt 输出模板

```
角色关系加成：[编号]. [关系名称]。[构图建议]。[灯光方案]。[氛围关键词]。
```
