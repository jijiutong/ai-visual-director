# 情绪曲线驱动分镜引擎

根据情绪弧线动态调整每个镜头的景别/运镜/灯光/色彩。数据表见 `knowledge/emotion-curves.md`。

---

## 决策逻辑

### 步骤1：识别情绪曲线类型

从 `state/variable-registry.md` 读取 `project.genre`（由 story-intake 写入），匹配曲线编号：

| 故事类型 | 曲线编号 |
|---------|----------|
| 动作/科幻/大多数商业 | EC1 标准四阶段 |
| 悬疑/犯罪/推理/心理惊悚 | EC2 悬疑五阶段 |
| 喜剧/搞笑/轻松 | EC3 喜剧四阶段 |
| 悲剧/虐心/宿命 | EC4 悲剧四阶段 |
| 爱情/甜宠/双向暗恋 | EC5 爱情五阶段 |
| 成长/冒险/热血/少年漫 | EC6 成长六阶段 |
| 复仇/武侠/黑帮 | EC7 复仇五阶段 |
| 拆弹/救援/限时任务 | EC8 压迫释放 |
| 心理/社会现实/霸凌 | EC9 压抑爆发 |
| 破镜重圆/前世今生/失忆重逢 | EC10 双线情感 |
| 智斗/悬疑策略 | EC11 反高潮 |
| 文艺/散文/生活流 | EC12 循环往复 |

> 完整匹配表见 `knowledge/emotion-curves.md` 五、类型→曲线匹配表

---

### 步骤2：获取镜数分配

根据曲线编号 + 镜数（来自 `engines/pacing.md`），查表获取每镜分配：

| 镜数 | 适用曲线 | 分配策略 |
|------|---------|---------|
| 3镜 | 所有曲线 | 压缩为：起点→高潮→余韵 |
| 5镜 | 所有曲线 | 去掉过渡阶段，保留核心三阶段 |
| 7镜 | EC1/EC2/EC4/EC5/EC8/EC9/EC11/EC12 | 按对应曲线表分配 |
| 9镜 | EC7 | 按EC7分配表 |
| 10镜 | EC6/EC10 | 按对应曲线表分配 |

> 完整分配表见 `knowledge/emotion-curves.md` 三、各曲线镜数分配表

---

### 步骤3：获取阶段参数

根据每镜的阶段，从 `knowledge/emotion-curves.md` 获取：
- 景别
- 运镜
- 灯光
- 色彩
- 节奏
- 构图

---

### 步骤4：输出分镜结构

整合曲线编号 + 镜数分配 + 阶段参数，输出完整分镜结构。

---

## 联动流程图

```
故事类型 ──→ emotion-curve.md ──→ 曲线编号(EC1-EC12)
                │
                ▼
           曲线 + 镜数 ──→ 每镜分配
                │
                ▼
        knowledge/emotion-curves.md ──→ 阶段参数
                │
                ▼
             输出分镜结构
```

---

## 与 pacing.md 联动

1. `pacing.md` 根据故事类型 → 确定节奏编号
2. `pacing.md` 根据节奏 + 时长 → 确定镜数
3. 本引擎根据故事类型 → 确定曲线编号
4. 本引擎根据曲线 + 镜数 → 确定每镜分配

---
