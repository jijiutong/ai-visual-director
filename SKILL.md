# AI Visual Director Skill

AI 电视剧 / 漫剧 / 短剧 / 电影视觉导演系统。将任意故事内容转化为角色卡、场景图、分镜图、视频 Prompt 和可执行视频生成包。

## 6 个核心入口

| 入口 | 定位 | 何时使用 |
|------|------|----------|
| `/create` | 一键总编排 | 用户给一个故事，想同时规划角色卡、场景图、分镜图和视频执行包 |
| `/source` | 输入源/项目读取 | Obsidian、Markdown、frontmatter、章节批量 |
| `/storyboard` | 核心资产：故事板/分镜图 | 全案板、分镜表、镜头设计、分镜图 Prompt |
| `/character` | 核心资产：角色卡 | 角色卡、三视图、表情、服装武器、角色 DNA |
| `/scene` | 核心资产：场景图 | 场景参考图、全景、空间关系、光照材质 |
| `/video` | 视频执行包 | 已有故事板/资产后，输出平台视频 Prompt 和执行清单 |

**兼容入口**：`/style` 和 `/poster` 继续可用，但定位为高级动作/输出格式，不再是主流程入口。

## 默认总路由

普通故事输入默认走 `/create` 编排链路。`/create` 不替代 `/storyboard`、`/character`、`/scene`，而是调用它们产出三大核心资产：

```text
角色卡（谁） + 场景图（在哪） + 分镜图（怎么拍） -> 视频 Prompt / 执行包
```

```text
task-router（意图识别+路由分发）
  -> sources/（输入源处理：粘贴/Obsidian/frontmatter）
  -> engines/story-intake.md          → 写入 state/variable-registry
  -> engines/shot-budget.md           → 写入 state/variable-registry
  -> engines/video-director.md        → 写入 state/variable-registry + shot-state + dialogue-map
  -> engines/asset-plan.md            → 写入 state/variable-registry
  -> engines/reference-anchor.md      → 写入 state/asset-map
  -> engines/motion-physics.md        → 补充 state/shot-state
  -> engines/project-graph.md         → 构建 state/project-graph（依赖图：正向索引+反向索引）
  -> engines/video-prompt-assembly.md ← 读取 state/
  -> engines/consistency-engine.md    → 5 维度 RM 评估（Character/Scene/Style/Story/Video）
  -> engines/prompt-scorer.md
  -> engines/auto-repair.md           → 修复后更新 state/（调用 knowledge-retrieval + incremental-update 限定重评）
  -> rules/final-video-qc.md          ← 读取 state/（9项质检含引用一致性+模板合规）
  -> engines/render-package.md
```

**基础能力子路由**：`/character` `/scene` `/storyboard` 三条标准子链，全部接 state/ + asset-map + QC。详见 `engines/task-router.md` → 基础能力子路由。

## 输入源

支持 2 种输入方式，自动识别无缝切换：

### 方式一：直接粘贴（默认）

用户直接把故事/剧本/小说段落粘贴到对话中。

### 方式二：Obsidian 读取（推荐长篇小说用户）

如果你的小说/剧本存在 Obsidian Vault 中，直接读，无需复制粘贴。

```
触发：从 Obsidian 读取 [章节名/笔记名]
     Obsidian 读取 [路径]
```

**支持的 Obsidian 结构**：

| 结构类型 | 说明 | 自动行为 |
|---------|------|---------|
| 单文件小说 | 一个笔记 = 整个故事 | 读取全文 → 提取信息 → 生成 |
| 章节文件夹 | `小说名/第1章.md` `小说名/第2章.md`... | 读取指定章节 | 
| 带 frontmatter 的笔记 | 笔记顶部 YAML 元数据标注了角色/场景/类型 | 自动解析元数据，跳过部分提取步骤 |
| 角色独立笔记 | `角色/张三.md` 含角色描述 | 自动读取角色笔记作为 DNA 初始值 |
| 场景独立笔记 | `场景/昆仑山.md` 含场景描述 | 自动读取场景笔记作为场景参考 |
| 全项目模式 | 一个文件夹里「小说.md + 角色/*.md + 场景/*.md」 | 一键读取全部关联笔记，交叉引用 |

**frontmatter 自动解析**（如果笔记有 YAML 头）：
```yaml
---
title: 剑道独尊
genre: 玄幻
style: 东方玄幻
characters: [["张三", "主角", "剑修"], ["李四", "反派", "魔修"]]
scene: 昆仑山剑冢
emotion: 悲壮
structure: 三幕
target_platform: Seedance
---
```
→ 这些字段直接映射到 Step 1 的提取字段，用户无需重复输入。

**读取长篇小说时的交互流程**：

  用户说「从 Obsidian 读取 [目录/文件夹名]」后，AI 先扫描，不直接生成：

  ```
  【Obsidian 项目扫描结果】
  
  📁 剑道独尊/
  ├── 第1章·剑冢觉醒.md (3200字)
  ├── 第2章·下山试剑.md (2800字)
  ├── 第3章·初入江湖.md (3400字)
  ├── ...
  ├── 第12章·决战天穹.md (4100字)
  ├── 角色/ (3个角色笔记)
  │   ├── 张三.md (主角)
  │   ├── 李四.md (反派)
  │   └── 王五.md (配角)
  └── 场景/ (2个场景笔记)
      ├── 剑冢.md
      └── 天穹殿.md
  
  请选择：
  1. 要处理哪些章节？
     回复编号，如「1-3」或「1,3,5」或「全部」
  
  2. 每章做成多少秒的视频？
     回复秒数，如「15s」或「30s」
     建议：短章节(＜2000字)=15s，中章节=30s，长章节=60s
  
  3. 每章需要多少分镜？
     回复镜数，如「7镜」或「10镜」
     建议：15s=7镜，30s=10镜，60s=15镜
  ```

  用户选择后，AI 批量生成该章节的故事板+Prompt+视频素材：
  - 章节1→15s→7镜→角色卡+场景图→分镜图→Prompt
  - 章节2→15s→7镜→跨段衔接→Prompt
  - ...
  - 自动跨章衔接（第N章尾帧 = 第N+1章首帧锚定）

  **快捷指令（跳过选择）**：
  - 「Obsidian 读取 剑道独尊 全部 15s 7镜」 → 全章批量，每章15s/7镜
  - 「Obsidian 读取 第1章 30s」 → 单章，30s，自动匹配镜数

## 工作流

```
输入故事 → 自动路由 → 输入源处理 → 故事摄入 → 镜头预算 → 视频导演 → 资产规划 → 锚点分配 → 运动检查 → 依赖图构建 → 平台适配 → 一致性评估 → 最终质检 → 修复 → 执行包
```

**快捷模式**：使用 `/create` 或自然语言 `一键生成`。`/storyboard` 仅作为分镜专用入口。

**Prompt 构建层级**（17层叠加）：

1. **基础层**：从模板文件获取格式结构
2. **风格层**：从 `engines/styles.md` 获取风格配色/氛围/特效（或 `engines/fusion.md` 融合风格）
3. **版式层**：从 `engines/layout-styles.md` 获取 LS 版式结构（全案/分镜/场景/角色卡）
4. **时代层**：从 `knowledge/historical-eras.md` 获取时代建筑/服饰/道具/礼仪/饮食精准描述
5. **关系层**：从 `engines/relationships.md` 追加角色关系加成
6. **结构层**：从 `engines/act-structure.md` 驱动三幕结构叙事
7. **情绪层**：从 `engines/emotion-curve.md` 驱动分镜节奏 + `engines/color-narrative.md` 驱动色彩叙事
8. **Mood 层**：从 `engines/mood-slider.md` 调整情绪权重/色彩/镜头/节奏
9. **DNA 层**：从 `knowledge/character-dna.md` 锚定角色一致性（20字段）
10. **生物/动物层**：从 `knowledge/creatures.md` + `knowledge/animals.md` 获取生物/动物外观/行为描述
11. **环境/道具层**：从 `knowledge/environments.md` + `knowledge/props.md` 获取场景/道具描述
12. **天气/材质层**：从 `knowledge/weather.md` + `knowledge/materials.md` 获取天气大气/材质质感
13. **身体语言/表情层**：从 `knowledge/body-language.md` + `knowledge/micro-expressions.md` + `knowledge/dialogue-rhythm.md` 获取姿态/表情/台词节奏
14. **转场/声音层**：从 `knowledge/transitions.md` + `knowledge/sound-design.md` 获取镜头转场/音乐音效
15. **文化层**：从 `rules/cultural-accuracy.md` 确保文化精准，杜绝穿帮
16. **负面层**：从 `rules/quality.md` + `rules/negative-prompt.md` 生成专属负面提示词
17. **平台层**：从 `platforms/platform.md` + `platforms/platform-advanced.md` 适配目标平台格式

  ─────────────────────────────────────────────

  ─────────────────────────────────────────────

【长视频跨段衔接 · 尾帧→首帧锁定】（多段视频拼接时使用，防止段落之间场景/角色瞬跳）

  原理：将上一段视频的最后 1-3 帧（导演决定取几帧），作为下一段视频的首帧参考图。
  AI 视频工具看到首帧 = 上段尾帧，自动平滑延续动作/装扮/场景，杜绝跳切。

  步骤：
  1. 生成第一段视频（如 15s 的前半部分）
     → 出片后，在视频末尾截取最后 1-3 帧（由导演决定：1帧=接力点，3帧=缓冲过渡）
     → 截取方式：播放器逐帧步进 / FFmpeg 精确截图 / 视频平台下载原始帧

  2. 尾帧转换为参考图（自动或手动）
     ```
     # FFmpeg 截取视频最后3帧
     ffmpeg -sseof -1 -i video_segment1.mp4 -vf fps=1 -vframes 3 tail_frame_%d.png
     ```

  3. 第二段视频的首帧 = 上段尾帧
     → Seedance：将尾帧图作为「首帧」参数输入
     → Runway：将尾帧图作为 promptImage 输入，加 prompt「continue from this exact frame，seamless」
     → 可灵：将尾帧图作为 first_frame media 输入

  4. 第二段 prompt 必须声明衔接约束
     ```
     prompt 开头追加：
     「从上一段视频的最后一帧无缝接续，场景/光线/角色位置/服装完全一致，
     动作从[上段结束动作]自然过渡到[本段开始动作]，禁止跳切硬切」
     ```
     prompt 英文尾追加：
     「seamless continuity from previous clip's last frame, continuous motion no jump cut」

  5. 生成第二段视频 → 检查衔接处是否平滑 → 如跳切则调整尾帧截取位置重新生成

  多段拼接（长片＞30s）：
  ```
  段1 → [截尾帧1-3] → 段2首帧锚定 → 段2 → [截尾帧1-3] → 段3首帧锚定 → ...
  每一段的尾帧都是下一段的首帧，形成平滑接力链
  ```

  尾帧截取策略（AI 读取故事板分段点的叙事上下文，自动匹配导演剪辑技法）：

  ── 视觉匹配类 ──

  · 图形匹配剪 — 上下段首尾帧有相同形状/轮廓/颜色（圆形→太阳、门框→窗口、刀弧→彩虹），视觉无缝对接
  · 运动矢量匹配 — 上段角色向右冲出画，下段角色/物体从左侧冲入，运动方向接力不中断
  · 视线匹配剪 — 上段角色看向画外某方向，下段直接切被看的人/物，视角从看→被看
  · 动作匹配剪 — 上段角色举刀，下段刀已落下（省略中间过程），同一动作跨段完成
  · 物件桥接剪 — 上段尾帧某物品特写，下段同一物品在不同时空出现，物品作跨段锚点
  · 光影衔接 — 上段尾帧有强光闪过（闪电/爆炸/开门光），利用瞬间过曝/全白遮蔽切段
  · 色彩洗入 — 上段尾帧逐渐偏向某色调，下段从同色调开始再还原，视觉情绪连贯
  · 焦点过渡 — 上段尾帧焦点变软/虚化，下段从模糊中聚焦到新画面，物理镜头感
  · 遮挡擦除 — 上段尾帧被人/物/墙完全遮挡画面瞬间切段，利用纯黑/纯色幕布效应
  · 晃动过渡 — 上段尾帧镜头剧烈晃动/撞击，下段从晃动中稳定下来（或反之），体感接力

  ── 叙事节奏类 ──

  · 动作高潮断 — 在挥砍/爆炸/撞击峰值帧截取，下一段从冲击瞬间或冲击后果开始
  · 情绪持留 — 在角色表情变化顶峰后持留 2-3 帧，让情绪沉下去再切，不让观众被硬拽走
  · 悬念切断 — 在疑问/威胁即将揭示前一帧截断，下一段从揭示开始，中间给观众呼吸间隙
  · 反差撞击 — 上段极静/极慢 → 下段极响/极快（或反过来），故意制造感官冲击
  · 喜剧节拍 — 在笑点 setup 的最后一帧截断，下一段直接给 punchline，利用时间差制造笑果
  · 时间跳跃 — 截取跳变前最后一帧，下段 prompt 声明时空已变，场景参考图可能部分切换
  · 梦境/幻觉过渡 — 上段尾帧开始出现超现实元素（色彩偏移/扭曲/重复），下段完全进入梦境状态
  · 记忆闪回 — 上段角色看到某物/听到某声触发回忆，尾帧拉近眼睛/物件特写，下段切入过去
  · 平行交叉剪 — 两段同一时刻不同空间来回切，每段尾帧为下一个空间的交替锚点
  · 倒计时加速 — 上段尾帧倒计时/钟表/沙漏，下段时间到/事件发生，时间压力驱动切段

  ── 声音驱动类 ──

  · J-Cut（声音先行）— 下段声音提前 0.5-2s 进入，上段画面还在但耳朵已到下一段
  · L-Cut（声音延留）— 上段声音延续到下段画面开始后 0.5-2s，声音作跨段纽带
  · 声音撞击 — 上段安静 → 下段巨响（或反之），利用音量差制造冲击力
  · 音乐短语完成 — 在音乐一个完整乐句/和弦解决处切段，不能卡在半句中间
  · 环境音渐变 — 上段环境音渐弱 + 下段环境音渐强交叉，听觉上无断崖
  · 静默过渡 — 上段尾帧去声 0.5-1s，短暂真空后下段声音入，利用沉默重置情绪

  ── 特殊技法类 ──

  · 冷冻帧过渡 — 上段尾帧定格为静帧，停留 0.5-1s 后下段开始，强调上段最后一刻
  · 速度渐变 — 上段尾帧逐渐慢动作→定格，下段从定格→逐渐恢复常速，时间感知扭曲
  · 倒放过渡 — 上段尾帧动作短倒放再正放切入下段，制造时间回弹感
  · 裂像转场 — 上段尾帧画面被"撕开"/"碎裂"，下段从裂口/碎片中浮现
  · 画幅变化 — 上段尾帧画幅比例开始变化（16:9→2.35:1 或反之），下段用新画幅，叙事空间切换
  · 分屏汇合 — 上段尾帧分成两个画面（分屏），下段两个画面汇合为一个完整画面，或多线合一
  · 色彩抽离 — 上段尾帧逐渐褪色到黑白，下段从黑白恢复色彩，用于区分时间/情绪/状态
  · 负片/反相过渡 — 上段尾帧色彩反转，下段恢复正常，用于进入幻觉/回忆/替代现实

  通用规则：
  → 默认 AI 自动分析故事板分段点，匹配最佳衔接技法（无需手动选择）
  → 也可手动指定：「用 J-Cut 衔接」「用图形匹配剪」「用冷冻帧过渡」— 强制使用指定技法
  → 尾帧永远是上段的最后帧，作为下段的首帧锚定参考图输入视频工具
  → 场景参考图/角色卡跨段复用，仅首帧锚定图更换为尾帧
  → Prompt 中声明衔接技法类型（如「J-Cut：下段环境音提前 1s 进入」「图形匹配：月亮→瞳孔」）

【AI 自动续写下一段剧情】（长视频多段拼接时，AI 基于当前段自动生成下一段故事板）：

  用户说「继续下一段」或「续写下一段」时，AI 自动：
  1. 读取当前段故事板的所有帧 → 分析当前段结束时的叙事状态（角色位置、情绪、动作方向、冲突阶段）
  2. 基于叙事结构（三幕/四幕/英雄之旅）自动推进：
     → 当前段是建置（第一幕）→ 下一段自动进入对抗（第二幕），冲突升级
     → 当前段是对抗中 → 下一段推进冲突到更深阶段（增加障碍/揭示真相/引入新角色）
     → 当前段是对抗高潮 → 下一段进入解决（第三幕），开始收束
  3. 延续所有锚点：
     → 角色 DNA 不变（服装可能因剧情变化而更新，标注可变字段）
     → 场景可能延续或切换（根据叙事推进判断是否需要新场景）
     → 情绪曲线按所选曲线模型（标准/悬疑/悲剧/爱情）自动推进到下一阶段
  4. 输出格式与当前段一致：N 帧故事板 + 技术参数 + 转场说明
  5. 用户确认后再生成该段的参考图/分镜图/prompt → 进入跨段衔接流程

  适用场景：
  → 长片分段制作（30s+的视频拆为多个 15s 段，AI 逐段续写）
  → 系列短剧（每集 15s，AI 自动推进剧情）
  → 不确定下一步怎么拍 → AI 给出 2-3 个剧情发展方向供选择

  连续性保障（AI 续写时强制检查，杜绝偏差歧义）：

  【硬锁定 · 不可变】（跨段绝对不允许变化，变了就是错误）：
  □ 角色 DNA 不可变字段：姓名/脸型/眼型瞳色/眉型/鼻型/嘴型/肤色/声音/气味
  □ 角色关系：已建立的角色关系不可反转（除非剧情明确写了背叛/反转等情节需要）
  □ 世界观规则：已设定的世界规则不可新增矛盾（如"这个世界没有魔法"→下一段不能突然出现魔法）
  □ 核心冲突：核心对抗目标不可变（如"争夺昆仑镜"→不能突然变成"争夺轩辕剑"）
  □ 时间线方向：时间不能往回走（除非明确标注闪回），段间时间间隔必须在 prompt 中声明

  【软锁定 · 可演进】（可变化但必须有说明）：
  □ 角色服装：可因战斗破损/场景需要而更换，必须在帧描述中标注变化原因
  □ 角色发型：可因战斗散开/被剪，必须标注
  □ 情绪状态：按情绪曲线推进，必须与上一段末尾情绪衔接，不跳级
  □ 场景：可切换但必须有过渡说明（离开→到达），同一场景复用必须场景参考图不变
  □ 道具/武器：可损坏/丢失/获得新道具，必须有获得或失去的因果交代
  □ 光线/天气：可变化但必须有原因（时间推移/天气变化/人工光源变化）

  【叙事逻辑检查】（续写内容必须通过以下检查）：
  □ 因果链完整：当前段结尾状态 → 下一段开头状态，中间发生了什么？如省略必须暗示
  □ 动机连贯：角色的下一步行动必须有上段铺垫的动机支撑，不凭空出现新动机
  □ 信息不跳级：角色掌握的信息不能突然多于上一段知道的信息（除非揭示了新的信息来源）
  □ 数字/数量一致：角色数/道具数/时间刻度不凭空增减
  □ 称呼一致：同一角色跨段称呼不变（名字/绰号/敬称不混用）

  【歧义检测】（发现以下情况暂停续写，询问用户或自动修正）：
  1. 角色动机断层 — 上段角色悲伤独处 → 下段突然兴高采烈战斗，中间无情绪过渡
     → 自动插入过渡镜或标注情绪转变原因
  2. 场景瞬移 — 上段在室内密谈 → 下段在战场厮杀，无离开/到达/时间推进
     → 必须补充「[X时间后]，[角色]到达[新场景]」
  3. 时间矛盾 — 上段标注"深夜" → 下段标注"正午"，中间无时间推进说明
     → 必须声明「次日正午」或「数小时后」
  4. 关系突变 — 上段两人敌对 → 下段突然联手，无和解过程
     → 插入转折事件说明或转为合作动机
  5. 能力凭空出现 — 上段角色未展示某能力 → 下段突然使用该能力
     → 必须在前段埋下伏笔或本段揭示来源
  6. 关键道具消失 — 上段角色手持某重要道具 → 下段道具不在手中且无交代
     → 标注道具状态或补充丢失情节
  7. 对话引用错误 — 下段角色复述上段对话时内容有偏差
     → 逐字核对上段台词
  8. 情绪数值跳级 — 按情绪曲线最多跳一个阶段（如平静→对峙），不能跨两阶段（平静→爆发）
     → 插入中间阶段或标注极端事件驱动

  续写输出前自检流程：
  1. 生成续写故事板草稿
  2. 逐项跑以上所有检查
  3. 发现硬锁定违反 → 直接拒绝，重新生成
  4. 发现软锁定变化但无说明 → 自动补充说明
  5. 发现歧义/逻辑矛盾 → 标注问题 + 给出修正方案
  6. 全部通过 → 输出完整续写故事板 + 衔接方案

【跨段衔接 · 选择模式】：
  → 自动模式（默认）：回复「出下一段」或「继续」，AI 自动分析故事板 + 匹配衔接技法
  → 手动指定衔接技法：「用[技法名]衔接，继续下一段」
    例：「用 J-Cut 衔接，继续下一段」
    例：「用图形匹配剪衔接，上段月亮→下段瞳孔，继续下一段」
    例：「用冷冻帧过渡衔接，继续下一段」
  → 手动指定剧情方向：「下一段往[方向]发展，用[技法]衔接」
    例：「下一段揭示反派身份，用悬念切断衔接」
    例：「下一段时间跳到三天后，场景不变但光线改黄昏，用时间跳跃衔接」
    例：「下一段双线并行，同时展示A在逃亡和B在追踪，用平行交叉剪衔接」

  场景一致性（跨段）：
  → 跨段必须复用同一张场景参考图（720° 全景图），锁死背景/空间
  → 角色卡跨段复用，锁死面部/体态
  → 仅首帧换（用上段尾帧），其余参考图不变
  → 详见 `rules/scene-consistency.md`

【传统流程（备选）】

 主参考：故事板（已有）— 视频的绝对主线，运镜/节奏/时间线/转场全部按故事板执行
  → 故事板包含 N 帧完整序列，是视频生成的唯一权威参考

▸ 辅助锚点 1：场景参考图 — 防止场景跑偏（环境/光线/天空/氛围跳变）
  → 回复"生成场景参考图"或"场景锚点"，可选以下 7 种方法（可叠加）：
    【基础方法】
    · 俯拍图 — 鸟瞰场景全貌，锁定空间布局/地形/建筑位置，适合固定场景
    · 4 宫格 — 场景 4 个关键角度（正/侧/背/近景），适合中小场景
    · 9 宫格 — 场景 9 个细节视角（全貌+局部+入口+关键元素+道具+光影），适合复杂场景
    · 默认 — 单张全景环境图
    【进阶方法】
    · 全能参考模式（推荐★）— 一张图同时锁死场景+角色+光照+道具，效率最高
      → 回复"全能参考图"或"all-in-one"
      → 角色占画面 1/3 站姿中央，场景全景为背景，标注主光方向+地面材质+天空色调
      → 适合：快速出片、单场景视频、Seedance 简略模式
    · 720° 全景图 — 360°×180° 全景环境，覆盖所有拍摄方向
      → 回复"生成全景图"或"720全景"
      → 方式 A：6 方向立方体贴图拼接（推荐）| 方式 B：实景拍摄 | 方式 C：直接生成（目前受限，主流模型不支持 equirectangular）
    · 环绕视频截图法 — 先生成环绕场景视频，再截取 8 个角度的关键帧
      → 回复"环绕截图"或"视频截图法"
      → 步骤：Seedance 生成 10-15s 环绕视频 → 截取 0°/45°/90°/.../315° 8 帧 → 拼为参考图
    · 文字场景锁定 · 多角度分镜 — 场景说明书作共同前缀，逐镜生成不同角度分镜图
      → 回复"镜头角度分镜"或"camera remix"
      → 步骤：写场景说明书（空间/光源/道具精确描述）→ 每镜 prompt = 说明书 + 镜头指令
      → 真正 img2img 改角度：推荐 ComfyUI ControlNet Depth 或 Nano Banana 多参考图
  → 详见 `rules/scene-consistency.md`
  → 作用：锁死背景/天空/环境布局跨帧一致不变

▸ 辅助锚点 2：光照材质参考图 — 防止光线方向/阴影/材质质感漂移
  → 回复"生成光照材质图"或"打光材质锚点"
  → 一张合成图，包含：
    · 左侧：光照方案示意（主光方向+辅光+轮廓光+色温标注+阴影方向）
    · 右侧：材质色板条（布料纹理/金属表面/皮肤质感/地面材质/水面反射各一小块色样）
  → 作用：锁死光线角度/阴影软硬/材质反光率/布料纹理跨帧不变

▸ 辅助锚点 3：道具细节卡 — 防止配饰/武器/小物件变形
  → 回复"生成道具细节卡"或"道具锚点"
  → 一张独立参考图：武器正面+侧面+细节特写 | 腰带/额饰/耳饰/挂件正面+侧面 | 红色标注不可变细节（纹路/比例/材质/颜色）
  → 作用：锁死道具形制/纹路/比例/颜色跨帧不变

▸ 辅助锚点 4：首尾帧 — 防止人物动作/装扮/面容在长镜头中漂移
  → 回复"生成首尾帧"或"首尾帧"
  → 生成帧 1（起始状态）和帧 N（结束状态）两张对比图
  → 作用：AI 视频工具根据首尾帧插值中间帧，确保人物从起始姿势→结束姿势平滑过渡，服装/面容/体态不漂移
  → 适合：变装转场/长时间镜头/动作幅度大的片段

▸ 辅助锚点 5：角色设定卡 — 防止角色跑偏（8 种方法可选，详见 `rules/character-consistency.md`）
  → 回复"生成角色卡"（默认 6 模块角色设定卡）
  → 进阶：回复"面部多角度"→ 5 角度面部特写 | "12表情"→ 表情范围图
  → 最强：回复"14图参考"→ Nano Banana 14 张参考图 | "IP-Adapter"→ ComfyUI 本地（社区扩展）
  → 零成本：回复"角色 DNA"→ 20 字段文字锚定
  → 作用：确保视频中角色面部/体态不变形

▸ 辅助锚点 6：视频分镜图（3-4 张）— 防止画面跑偏
  → 回复"生成视频分镜图"或"合并帧"
  → 将故事板 N 帧合并为 3-4 张（每张覆盖 2-3 帧），按动作阶段分组
  → 作用：给 AI 视频工具关键视觉锚点，不偏离故事板

 逐帧接力（备选，精细控制）
  → 回复"切分各帧"或"逐帧接力"，逐帧输出（N 帧 = N 张图）
  → 帧间参考链：帧1 先生成 → 帧1 作参考图出帧2 → 帧2 作参考出帧3 → ... 链式传递
  → 适用于需要逐帧精确控制的场景

【视频 Prompt 输出格式】
- 画面内容：从故事板 N 帧画面描述逐帧提取，只保留核心视觉信息（角色动作+环境），不超过 15 字
- 运镜/焦段/角度：从故事板技术参数栏逐帧读取，用 `|` 分隔
- 色彩/灯光：从故事板技术参数栏逐帧读取，用 `|` 分隔
- 转场/时长：从故事板技术参数栏逐帧读取，用 `|` 分隔
- 场景一致性：引用场景参考图（全景图或全能参考图，覆盖所有拍摄方向，背景/天空/空间布局跨帧不变）
- 角色一致性：引用角色卡（面部5角度/表情范围/三视图/服装武器细节，面部/体态/服装不变形）
- 画面锚点：引用视频分镜图（按动作阶段合并，3-4张，防视觉偏离，每张覆盖2-3帧）
- （可选）光照材质一致性：额外光照材质参考图（主光方向/阴影软硬/色温/材质反光率跨帧不变）
- （可选）道具一致性：额外道具细节卡（武器形制/配饰纹路/挂件比例/颜色跨帧不变）
- （可选）人物动作一致性：额外首尾帧（帧1起始→帧N结束平滑过渡，长镜头/动作大片建议加）
- ⚠️ 具体 @编号→用途 映射由 `state/asset-map.md` 动态决定（按平台策略：Seedance 5-6张、Runway 3张、可灵 3张、Luma/Pika 2张），禁止硬编码 @图N=某用途
- 台词处理：如故事板帧含台词，帧描述中简写「台词「[内容]」[表演要求]」；约束层写「台词：帧 N 口型同步「[内容]」[表演要求]。字幕：帧 N 底部中央「[内容]」」
- 过渡衔接：每帧末尾指定过渡方式（渐切/慢动作过渡/叠化/淡出/淡入），禁止硬切闪跳；开头声明「整体一条连续镜头，帧与帧之间平滑过渡，不生硬跳转」；英文尾加 `continuous single shot, smooth transitions between frames no hard cuts`
- 每帧压缩为单行，技术参数用 `|` 分隔
- Prompt 总字数控制在 1500 字以内，最大可能遵从故事板，少写冗余文字
- 一条完整 prompt，直接喂 Seedance 出完整视频

【详细模式】（回复"详细模式"或"展开 prompt"，适用于 Runway/Sora/可灵等文生视频工具）：

## 执行规则

- **先提取，再生成** — 不要跳过故事信息提取
- **从 Obsidian 读取 [目录名]** → 扫描目录结构 → 展示章节/角色/场景清单 → 用户选章节+时长+分镜数 → 批量生成。快捷：`Obsidian 读取 [目录] 全部 [时长] [镜数]`
- **Obsidian 读取全部** → 读取整个文件夹（小说+角色+场景）→ 逐章批量生成
- **使用 /storyboard 生成故事板。智能推荐优先** — 默认展示 2-3 个推荐组合，用户确认即可
- **推荐不展示内部编号** — 默认推荐用「风格名+格式名」，如「东方玄幻全案板」，不用「3. 东方玄幻 + 1. 全案板」。风格编号只在用户说"看全部"时展示
- **每个括号都填实** — 不写"电影感""高级感"等空泛词
- **风格不混** — 一次只选一种视觉风格（融合模式除外）
- **用户说"更燃/更虐/更甜/更丧"** → 触发 Mood 滑块，调整情绪/色彩/镜头/节奏
- **用户说"更贵"** → 强化摄影参数/材质/色卡/版式秩序
- **用户说"小红书竖版"** → 画幅改 9:16，结构不变
- **用户说"看全部"** → 展示完整 53 种风格 + 10 种格式列表
- **用户说"第X集"/"继续"** → 触发续集模式，延续DNA/色彩/场景
- **关系加成** → 自动识别角色关系，追加构图/灯光/风格加成/氛围描述
- **情绪驱动** → 根据情绪曲线动态调整景别/运镜/灯光/色彩
- **DNA 锚定** → 生成角色 DNA 确保跨镜头一致性
- **文化精准** → 确保东方/西方/日系描述不穿帮
- **负面词自动** → 按风格/格式/关系/平台自动生成专属负面提示词
- **单镜修改** → "第X镜暗一点/换长发/雨改雪" 只改指定镜，通过项目依赖图自动计算影响范围并跨镜传播
- **单镜修改强制保留时间码** — 精准修改第 X 镜时，必须保留原镜号、原时间码、原镜头顺序。除非用户明确要求调整时长，否则不得改动时间码
- **连续性修改自动补桥** — 修改天气、时间、地点、角色外观等连续性强字段时，必须自动处理前后镜衔接（如雨转雪用"雨夹雪"过渡镜），并输出连续性提示说明影响范围
- **音乐音效** → 每幕/每镜自动推荐音乐 mood + 音效
- **缩略速出** → 先出关键词版验证方向，确认后再出完整版
- **导演阐述** → 追加镜头/色彩/角色弧光/节奏的选择理由
- **行业导出** → "导出 Storyboard Pro/FrameForge/Celtx" 标准格式
- **转视频** → 故事板→Sora/Runway/可灵/Seedance/Luma，带运动方向和时长
- **视频工作流** → 推荐流程：角色+场景+道具（2-3张）→ 全景图 → 分镜图（3-4张）→ prompt → 视频。不推荐仅故事板直接出视频
- **长视频跨段衔接 / 尾帧锁定** → 上段视频截取最后1-3帧作为下段首帧参考图，场景/角色/道具参考图跨段复用，prompt 声明无缝接续
- **多段拼接** → 段1→尾帧→段2→尾帧→段3... 每一段的尾帧 = 下一段的首帧，形成平滑接力链
- **继续下一段 / 续写下一段** → AI 自动分析当前段叙事状态，基于叙事结构推进，生成下一段故事板。延续所有锚点 + 自动匹配衔接技法
- **手动指定衔接技法** → 「用[技法名]衔接，继续下一段」— 强制使用指定技法而非自动匹配
- **手动指定剧情方向** → 「下一段往[方向]发展，用[技法]衔接」— 同时指定剧情走向和衔接技法
- **生成场景参考图 / 场景锚点** → 7 种方法可选（俯拍图/4宫格/9宫格/单张全景/全能参考/720全景/环绕截图/文字场景锁定），详见场景一致性参考
- **全能参考图 / all-in-one** → 一张图锁定场景+角色+光照+道具，最高效锚点方案
- **生成全景图 / 720全景** → 360°×180° 全景环境图，覆盖所有拍摄方向
- **环绕截图 / 视频截图法** → 生成环绕视频→截取8角度关键帧→拼接参考图
- **镜头角度分镜 / camera remix** → 文字场景锁定：场景说明书+镜头指令逐镜生成；真正 img2img 用 ComfyUI ControlNet 或 Nano Banana
- **生成光照材质图 / 打光材质锚点** → 一张合成图：光照方案（主光+辅光+轮廓光+色温+阴影方向）+ 材质色板条（布料/金属/皮肤/地面/水面各一色样），锁死光线角度/阴影/反光率/纹理
- **生成道具细节卡 / 道具锚点** → 武器+配饰+挂件多角度特写，红色标注不可变细节（纹路/比例/材质/颜色），锁死道具形制跨帧
- **生成首尾帧 / 首尾帧** → 帧1起始+帧N结束对比图，锁定动作不漂移
- **角色卡剧情身份** → 当故事存在双人/多人关系时，角色卡必须包含【剧情身份】字段（师父/徒弟/对手/旁观者）。若不明确，标注"身份待确认"，不得擅自改变原有关系
- **一键生成必须含锚点** → /storyboard 一键生成必须在分镜表前输出【角色锚点】（主角/对手简要 DNA）和【场景锚点】（场景名+空间+光照+氛围），确保后续每一镜可继承
- **三视图 / 角色三视图** → 正/侧/背三视图并排，最精简角色锚定
- **面部多角度 / 5角度** → 正面/左右45°/左右侧面 5 角度面部特写，专攻面部一致性
- **表情范围图 / 12表情** → 3×4 宫格 12 种表情，锁定情绪表达范围防变脸
- **服装武器细节卡** → 服装+武器独立细节参考，红标不可变细节
- **Nano Banana 角色锚定 / 14图参考** → 12-14 张角色参考图输入 Nano Banana，最强角色一致性
- **IP-Adapter 角色锁定** → ComfyUI IP-Adapter 本地零成本无限生成，角色特征向量注入
- **角色 DNA / DNA 锚定** → 20 字段角色文字描述，prompt 共用前缀，零成本保底
- **切分各帧 / 逐帧接力** → 将多帧合成故事板拆为逐帧独立 prompt，前一帧作后一帧参考图链式传递，每帧 16:9 单独出图
- **生成视频分镜图 / 合并帧** → 多帧故事板转视频时，按动作阶段合并相邻帧（如7帧压成3张），每张覆盖2-3帧，作为画面锚点防止跑偏
- **出视频 prompt** → 默认压缩模式（≤1500字），整合故事板全序列+所有锚点约束，直喂 Seedance
- **详细模式 / 展开 prompt** → 切换为详细模式（3000+字），每帧完整叙事段落+音效+构图+转场+台词，适合 Runway/Sora/可灵
- **视频负面提示词** → 自动附加到所有视频 prompt 尾部，防闪烁/扭曲/漂移/跳色/断连续性
- **检查视频 / 视频检查** → 出片后逐项打勾，场景/角色/光照/道具/动作/画面/转场/字幕/材质 10 项，定位问题归属
- **用 [平台] 生成 [格式]** → 调 API 直接出图（Flux/Ideogram/通义万相/ComfyUI/Recraft），需先配置 API Key
- **用 Seedance 生成视频** → 调火山引擎 Ark API，多图输入+中文原生+音频，异步轮询模式
- **用 Runway 生成视频** → 调 Runway Gen-4.5 API，SDK 完善，支持图生视频/文生视频
- **用 可灵 生成视频** → 调阿里云百炼 DashScope API，4K 输出，角色 ID 复用
- **用 Luma 生成视频** → 调 Luma Dream Machine API，首尾帧插值，丝滑过渡
- **用 Pika 生成视频** → 调 fal.ai Pika API，社交短视频，快速验证
- **视频 API** → 所有视频 API 为异步模式（创建任务→轮询→下载），需使用 `node_repl` MCP 执行
- **质量评分** → 0-100 分自动评估，低于 75 自动优化
- **连续性检查** → 自动检查时间线/空间/角色/情绪连贯性，标注断裂点
- **多画幅** → "适配小红书/抖音/朋友圈/所有平台" 自动切换画幅
- **版本管理** → "回滚到第2版/对比v1和v3/看版本历史" 管理修改记录
- **风格迁移** → "换成X风格但保持角色" 跨风格迁移只改变视觉语言（色彩/光影/构图/运镜/节奏/表演方式），绝对不改变剧情事实（剧情事件/人物关系/冲突结果/镜头数量/总时长/关键动作）
- **Prompt 压缩** → "压缩到MJ/SD" 自动精简到平台最佳长度
- **批量处理** → "处理这一章" 整章小说自动拆场景批量生成
详细设计见 `engines/README.md`（主链引擎 14 个 + state/ 9 个状态文件）和 `state/README.md`（状态生命周期）
