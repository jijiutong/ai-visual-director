# Obsidian Workflow — 长篇批量工作流

> 从 Obsidian Vault 读取小说 → 逐章批量生成故事板。
> 本功能模拟目录文本输入验证，不承诺真实 Vault 文件系统读取。

---

## 支持结构

| 结构类型 | 说明 | 自动行为 |
|---------|------|---------|
| 单文件小说 | 一个笔记 = 整个故事 | 读取全文 → 提取信息 → 生成 |
| 章节文件夹 | `小说名/第1章.md` `第2章.md`... | 按章节列表生成 |
| 带 frontmatter 的笔记 | YAML 元数据标注角色/场景/类型 | 自动解析，跳过部分提取 |
| 角色独立笔记 | `角色/张三.md` | 作为 DNA 初始值 |
| 场景独立笔记 | `场景/昆仑山.md` | 作为场景参考 |
| 全项目模式 | `小说.md + 角色/*.md + 场景/*.md` | 一键读取全部，交叉引用 |

---

## 交互流程

### 扫描阶段

```
用户: 从 Obsidian 读取 novel/

AI: 【Obsidian 项目扫描结果】

📁 novel/
├── 01-雨夜古寺.md (~500字)
├── 02-师门旧事.md (~480字)
├── 03-入魔真相.md (~520字)
├── characters.md (角色档案)
└── settings.md (场景档案)

请选择：
1. 要处理哪些章节？回复编号，如「1-3」或「全部」
2. 每章多少秒？建议：短章15s，中章30s
3. 每章多少分镜？建议：15s=7镜，30s=10镜
```

### 快捷指令（跳过选择）

```
Obsidian 读取 novel/ 全部 15s 7镜  → 全章批量
Obsidian 读取 novel/ 第1章 30s     → 单章
```

## Frontmatter 解析

笔记含 YAML 头时自动解析：

```yaml
---
title: 剑道独尊
genre: 武侠
style: 东方玄幻
characters: [["墨渊", "徒弟", "入魔"], ["顾长空", "师父"]]
scene: 雨夜古寺大殿
emotion: 悲壮
structure: 三幕
---
```

→ 字段直接映射到 Step 1（故事信息提取），无需重复输入。

## 批量输出

逐章生成 15s × 7镜 故事板：

```
章1 → 角色卡+场景图 → 分镜图 → Prompt
章2 → 跨章衔接（章1尾帧=章2首帧） → 分镜图 → Prompt
章3 → 跨章衔接（章2尾帧=章3首帧） → 分镜图 → Prompt
```

## 系列续写

### 第X集 / 继续

```
【系列延续】《剑道独尊》第2集
延续第1集设定：风格 东方玄幻，色彩基调 暗蓝灰#2a3a4a+烛火暖黄
角色 DNA 引用：墨渊（黑衣/长发/入魔）顾长空（灰白鬓发/蓑衣）
本集变化：场景→昆仑山门（闪回），情绪→温暖→愤怒
```

### 续写约束

**硬锁定**（不可变）：
- 角色 DNA 不可变字段
- 角色关系（除非剧情明确反转）
- 世界观规则
- 核心冲突
- 时间线方向

**软锁定**（可演进须说明）：
- 服装/发型/道具
- 情绪状态（按曲线推进，不跳级）
- 场景切换（有过渡说明）
- 光线/天气（有原因）

**叙事逻辑**：因果链完整、动机连贯、信息不跳级、称呼一致。

## 处理这一章

自动拆分单章为场景/回合：

```
原文 → AI识别场景切换点 → 拆分为3-5个场景 → 每个场景生成故事板

场景关系标注：场景1→场景2（动作连续/情绪升级/时间推进）
```

---

## 使用建议

| 场景 | 推荐方式 |
|------|---------|
| 一整个长篇 | Obsidian 读取 全部 15s 7镜 |
| 只想看某章 | Obsidian 读取 第3章 |
| 追更/周更 | 第X集 / 继续 |
| 不确定怎么推进 | 下一段往 [方向] 发展 |
| 有大量角色笔记 | 全项目模式（小说+角色+场景全读） |
