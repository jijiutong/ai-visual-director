# Obsidian 输入引擎

从 Obsidian Vault 读取小说/剧本项目，自动扫描章节结构、解析 frontmatter、交叉引用角色/场景笔记，输出统一故事结构给 `story-intake`。

---

## 触发方式

| 用户说法 | 行为 |
|---------|------|
| "从 Obsidian 读取 [路径]" | 扫描目录，列出内容 |
| "Obsidian 读取 [项目名] 全部 15s 7镜" | 全章批量，跳过确认 |
| "Obsidian 读取 [章节名] 30s" | 单章处理 |

---

## 扫描流程

### 1. 定位 Vault

```
1. 读取 api-config.template.env → OBSIDIAN_VAULT_PATH（默认 ~/Documents/Obsidian）
2. 用户可在配置中修改路径，或对话中说"Obsidian 在 [路径]"
3. 列出 Vault 直接子目录（项目文件夹）
4. 如用户指定了项目名 → 直接进入该项目目录
```

### 2. 扫描项目结构

进入项目目录后：

```
□ 列出所有 .md 文件
□ 识别结构模式：
  - 单文件模式：只有1个 .md → 就是整个故事
  - 章节文件夹模式：多个 .md 且文件名含数字 → 章节列表
  - 全项目模式：有子目录"角色"/"场景"/"设定" → 关联笔记
□ 统计每个文件字数
```

### 3. 输出扫描结果

```markdown
【Obsidian 项目扫描结果】

📁 [项目名]/  (路径: /Users/.../Obsidian/[项目名]/)

├── 第1章·[标题].md (3200字)
├── 第2章·[标题].md (2800字)
├── ...
├── 第N章·[标题].md (4100字)

[如有子目录]
├── 角色/ (N个角色笔记)
│   ├── [角色A].md
│   └── [角色B].md
└── 场景/ (N个场景笔记)
    ├── [场景A].md
    └── [场景B].md

请选择：
1. 要处理哪些章节？
   回复编号，如「1-3」或「1,3,5」或「全部」

2. 每章做成多少秒的视频？
   回复秒数，如「15s」或「30s」
   建议：由 video-director 多维度决策（叙事复杂度/情绪节拍密度/动作密度/场景切换频率/对话比例/高潮强度），最终受平台 {PLATFORM}_MAX_DURATION 上限约束

3. 每章需要多少分镜？
   回复镜数，如「7镜」或「10镜」
   建议：由 video-director 根据 pacing 类型和时长自动决定
```

## 关联笔记读取

### 角色笔记

如果项目有 `角色/` 子目录：

```
1. 读取所有角色 .md 文件
2. 提取角色名→描述映射
3. 在生成 story-intake 时，自动填充角色 DNA 初始值
4. 如果角色笔记含 frontmatter → 优先用 frontmatter 字段
```

### 场景笔记

如果项目有 `场景/` 子目录：

```
1. 读取所有场景 .md 文件
2. 提取场景名→描述映射
3. 在生成 story-intake 时，自动填充场景/世界观字段
4. 匹配规则：章节中提到的地名 → 场景笔记中的描述
```

### 交叉引用

章节正文中提到角色名/场景名时：
→ 自动关联到对应笔记的描述
→ 一次性输出：章节内容 + 角色描述 + 场景描述

---

## 批量处理

用户选择多个章节后：

```
对每个章节：
  1. 读取完整内容
  2. 关联角色/场景笔记
  3. 送入 story-intake（→ 写入 state/variable-registry）
  4. 送入 video-director（→ 写入 state/shot-state + state/dialogue-map）
  5. 生成故事板 + Prompt
  6. 如果是第N章(N>1) → 跨段衔接（读取上一章尾帧状态 ← state/shot-state end_state）

输出顺序：
  章节1 → 完成（角色卡+场景卡+分镜图+Prompt）
  章节2 → 跨段衔接 → 完成
  ...
```

---

## 快捷指令

| 指令 | 等价操作 |
|------|---------|
| `Obsidian 读取 [项目] 全部 15s 7镜` | 全章批量，每章15s/7镜 |
| `Obsidian 读取 [项目] 第1章 30s` | 单章，30s，自动匹配镜数 |
| `Obsidian 读取 [项目] 1-5 30s` | 第1-5章，每章30s |

---

## 输出

处理后送入 `story-intake` → 写入 `state/variable-registry`，随后进入主链或基础能力子路由。见 sources/README.md 统一输出格式。
