# 直接粘贴输入引擎

处理用户直接粘贴到对话中的故事/剧本/小说。

---

## 识别规则

| 输入特征 | 判断 | 处理方式 |
|---------|------|---------|
| 短句（≤100字） | 一句话故事 | 直接送 story-intake，自动补全 |
| 段落（100-500字） | 单个场景/短篇 | 送 story-intake 提取字段 |
| 长篇（500-2000字） | 完整短篇/章节 | 先压缩再送 shot-budget |
| 超长（>2000字） | 长篇章节/剧本 | 标记"需拆段"，触发分章 |

---

## 处理规则

### 短句模式

用户："雨夜古寺，两名剑客在佛像前对峙"
→ 不追问细节
→ 直接送 `story-intake` 自动补全（起名、定类型、推断冲突）→ 写入 `state/variable-registry`
→ 进入一键生成最快路径

### 段落模式

用户粘贴一段完整场景描述
→ 提取：主角/对手/场景/冲突/情绪
→ 如果元素不明确 → 最多追问1次（类型+时长）
→ 剩下的自动推断

### 长篇模式

用户粘贴一章小说
→ 标记字数→判断是否压缩
→ 调用 `shot-budget` 压缩
→ 如果用户没指定时长→按字数推断：
  - 500-1000字 → 30s
  - 1000-2000字 → 60s或拆段

---

## 缺字段自动补全

| 缺什么 | 怎么补 |
|--------|--------|
| 没角色名 | 按类型起电影感名字（武侠=诗意，科幻=科技感） |
| 没明确类型 | 按冲突强度→强冲突=动作，弱冲突=剧情，无冲突=文艺 |
| 没明确时长 | 按字数→短=15s，中=30s，长=60s或拆段 |
| 没明确场景 | 提取第一个出现的地点 |
| 没明确情绪 | 按冲突类型→对抗=紧张，离别=悲伤，重逢=温暖 |

---

## 输出

直接粘贴处理后输出统一故事结构（见 sources/README.md 统一输出格式），之后送入 `story-intake` → 写入 `state/variable-registry`。
