# Step 指南 — 完整工作流

从故事输入到视频执行包的完整 4 步流程。由根 `SKILL.md` 默认总路由驱动，`engines/task-router.md` 分发。

---

## 工作流总览

```
输入故事 → 自动路由 → 输入源处理 → 故事摄入 → 镜头预算 → 视频导演 → 资产规划 → 一致性触达 → 视觉密度 → 锚点分配 → 运动检查 → 依赖图构建 → 平台适配 → 一致性评估 → 最终质检 → 修复 → 执行包
```

**快捷模式**：使用 `/avd/create` 或自然语言 `一键生成`。`/avd/storyboard` 仅作为分镜专用入口。

---

## Step 1: 提取故事信息

从用户输入中自动提取：

| 字段 | 说明 | 参考文件 |
|------|------|---------|
| 片名 | 2-8 字短标题 | — |

| Mood 倾向 | 更燃/更虐/更甜/更丧/更暧昧 | `engines/mood-slider.md` |
| 镜头数 | 由 video-director + consistency-trigger 决策 | `engines/emotion-curve.md` |

**缺字段自动补全**：没角色名就给起电影感名字；没明确类型就按冲突强度推断；没明确关系就按互动张力推断；没明确结构就按类型匹配三幕结构。

### 视频工作流指引

> 以下为生成故事板后的标准视频制作流程。

**推荐流程 ★ 前置准备 → 全景锁定 → 分镜图 → Prompt → 视频**：

**第一步：定义角色+场景+道具（2-3 张参考图）**
→ 回复"生成角色卡"（角色设定卡，6 模块）
→ 回复"生成场景参考图"（选全能参考模式，一张锁场景+角色+光照+道具）
→ 回复"生成道具细节卡"（如有特殊武器/配饰）

**第二步：全景图锁定全局空间（720° 全景图）**
→ 回复"生成全景图"（推荐方式B：6面立方体拼接）
→ 全景图是所有后续分镜图的空间一致性基础

**第三步：生成分镜图**
→ 回复"生成视频分镜图"或"合并帧"
→ 将故事板 N 帧合并为 3-4 张（每张覆盖 2-3 帧），按动作阶段分组

**第四步：生成视频 prompt**
→ 回复"出视频 prompt"（默认压缩模式 ≤ 平台上限字）
→ AI 整合故事板全序列 + 角色卡 + 全景图 + 分镜图所有锚点 → 一条完整 prompt

**第五步：输入视频工具生成视频**
→ Seedance：prompt + 角色卡 + 全景图 + 分镜图

### 传统流程（备选）

**主参考**：故事板（已有）— 视频的绝对主线

**辅助锚点 1：场景参考图** → 7 种方法可选，详见 `rules/scene-consistency.md`
**辅助锚点 2：光照材质参考图** → 一张合成图：光照方案+材质色板
**辅助锚点 3：道具细节卡** → 武器+配饰多角度特写，红标不可变细节
**辅助锚点 4：首尾帧** → 帧1起始+帧N结束对比图
**辅助锚点 5：角色设定卡** → 8 种方法可选，详见 `rules/character-consistency.md`
**辅助锚点 6：视频分镜图（3-4 张）** → 按动作阶段合并

**逐帧接力（备选，精细控制）**：
→ 回复"切分各帧"或"逐帧接力"，逐帧输出
→ 帧间参考链：帧1→帧2→帧3...链式传递

### Seedance 多图上传顺序

**完整版（7 张）**：
```
1. 故事板合成图 — 首位，定义视频运镜/节奏/时间线
2. 全景图（720° 或全能参考图）— 锁死全部拍摄方向的背景/天空/空间布局
3. 角色设定卡（6 模块）— 锁死面部/体态/服装/武器
4-7. 视频分镜图 3-4 张 — 画面锚点，按帧序排列
```

**简略版（3 张）**：
```
1. 故事板合成图 — 主参考
2. 全能参考图（场景+角色+光照+道具一张搞定）
3. 视频分镜图（合并为 1 张）
```

### 视频 Prompt 输出格式

- 画面内容：从故事板 N 帧画面描述逐帧提取，只保留核心视觉信息，不超过 15 字
- 运镜/焦段/角度：从故事板技术参数栏逐帧读取，用 `|` 分隔
- 色彩/灯光：从故事板技术参数栏逐帧读取，用 `|` 分隔
- 转场/时长：从故事板技术参数栏逐帧读取，用 `|` 分隔
- @图映射：由 `state/asset-map.md` 动态决定（按平台策略）
- 台词处理：帧描述中简写「台词「[内容]」[表演要求]」
- 过渡衔接：每帧末尾指定过渡方式，禁止硬切闪跳
- 每帧压缩为单行，技术参数用 `|` 分隔
- Prompt 总字数控制在平台上限以内

**详细模式**（回复"详细模式"或"展开 prompt"，适用 Runway/Sora/可灵）：
- 每帧展开为完整段落，沉浸式叙事描述，不压缩字数
- 每帧包含：视频基础设定 → 分镜N → 景别 → 画面内容 → 运镜 → 构图 → 切换 → 台词 → 音效

## 跨段衔接 · 尾帧→首帧锁定

多段视频拼接时使用，防止段落之间场景/角色瞬跳。

**原理**：上一段视频最后 1-3 帧作为下一段视频的首帧参考图。AI 视频工具看到首帧=上段尾帧，自动平滑延续。

### 36 种衔接技法

**视觉匹配类（10 种）**：
图形匹配剪 / 运动矢量匹配 / 视线匹配剪 / 动作匹配剪 / 物件桥接剪 / 光影衔接 / 色彩洗入 / 焦点过渡 / 遮挡擦除 / 晃动过渡

**叙事节奏类（10 种）**：
动作高潮断 / 情绪持留 / 悬念切断 / 反差撞击 / 喜剧节拍 / 时间跳跃 / 梦境/幻觉过渡 / 记忆闪回 / 平行交叉剪 / 倒计时加速

**声音驱动类（6 种）**：
J-Cut（声音先行）/ L-Cut（声音延留）/ 声音撞击 / 音乐短语完成 / 环境音渐变 / 静默过渡

**特殊技法类（10 种）**：
冷冻帧过渡 / 速度渐变 / 倒放过渡 / 裂像转场 / 画幅变化 / 分屏汇合 / 色彩抽离 / 负片/反相过渡 / 多段拼接接力链

> 默认 AI 自动分析故事板分段点，匹配最佳衔接技法。也可手动指定：「用 J-Cut 衔接」「用图形匹配剪」等。

### AI 自动续写下一段剧情

用户说「继续下一段」或「续写下一段」时，AI 自动：

1. 读取当前段故事板 → 分析叙事状态
2. 基于叙事结构自动推进（建置→对抗→解决）
3. 延续所有锚点（角色DNA/场景/情绪曲线）
4. 输出格式与当前段一致
5. 用户确认后生成参考图/分镜图/prompt → 进入跨段衔接

**连续性保障（硬锁定/软锁定/叙事逻辑/歧义检测）**：
- **硬锁定不可变**：角色DNA/关系/世界观规则/核心冲突/时间线方向
- **软锁定可演进**：服装/发型/情绪/场景/道具/光线（变化必须有说明）
- **叙事逻辑检查**：因果链/动机/信息/数量/称呼 不跳级不矛盾
- **歧义检测 8 种**：动机断层/场景瞬移/时间矛盾/关系突变/能力凭空出现/道具消失/对话引用错误/情绪跳级

## 执行规则

- **先提取，再生成** — 不要跳过故事信息提取
- **从 Obsidian 读取 [目录名]** → 扫描 → 展示清单 → 用户选 → 批量生成
- **智能推荐优先** — 默认展示 2-3 个推荐组合，用户确认即可
- **推荐不展示内部编号** — 用「东方玄幻全案板」，不用「3. 东方玄幻 + 1. 全案板」
- **每个括号都填实** — 不写"电影感""高级感"等空泛词
- **风格不混** — 一次只选一种视觉风格（融合模式除外）
- **用户说"更燃/更虐/更甜/更丧"** → 触发 Mood 滑块
- **用户说"更贵"** → 强化摄影参数/材质/色卡/版式秩序
- **用户说"小红书竖版"** → 画幅改 9:16，结构不变
- **用户说"看全部"** → 展示完整 53 种风格 + 10 种格式
- **用户说"第X集"/"继续"** → 触发续集模式
- **关系加成** → 自动识别角色关系，追加构图/灯光/风格加成/氛围描述
- **情绪驱动** → 根据情绪曲线动态调整景别/运镜/灯光/色彩
- **DNA 锚定** → 生成角色 DNA 确保跨镜头一致性
- **文化精准** → 确保东方/西方/日系描述不穿帮
- **负面词自动** → 按风格/格式/关系/平台自动生成专属负面提示词
- **单镜修改** → "第X镜暗一点/换长发/雨改雪" 只改指定镜，自动计算影响范围
- **单镜修改强制保留时间码** — 保留原镜号、原时间码、原镜头顺序
- **连续性修改自动补桥** — 修改天气/时间等连续字段时，自动处理前后镜衔接
- **音乐音效** → 每幕/每镜自动推荐音乐 mood + 音效
- **缩略速出** → 先出关键词版验证方向，确认后再出完整版
- **导演阐述** → 追加镜头/色彩/角色弧光/节奏的选择理由
- **行业导出** → "导出 Storyboard Pro/FrameForge/Celtx" 标准格式
- **转视频** → 故事板→Sora/Runway/可灵/Seedance/Luma，带运动方向和时长
- **视频工作流** → 角色+场景+道具 → 全景图 → 分镜图 → prompt → 视频
- **长视频跨段衔接 / 尾帧锁定** → 上段尾帧=下段首帧，参考图跨段复用
- **多段拼接** → 段1→尾帧→段2→尾帧→段3... 形成平滑接力链
- **继续下一段 / 续写下一段** → AI 自动分析+推进+生成+衔接
- **一键生成必须含锚点** → 必须输出角色锚点和场景锚点
- **出视频 prompt** → 默认压缩模式（≤ 平台上限字）
- **详细模式 / 展开 prompt** → 完整段落（3000+字）
- **检查视频 / 视频检查** → 出片后 12 项逐项打勾
- **质量评分** → 0-100 分自动评估，低于 75 自动优化
- **连续性检查** → 自动检查时间线/空间/角色/情绪连贯性
- **多画幅** → "适配小红书/抖音/朋友圈/所有平台" 自动切换
- **版本管理** → "回滚到第2版/对比v1和v3/看版本历史"
- **风格迁移** → "换成X风格但保持角色" 只改变视觉语言，不改变剧情事实
- **Prompt 压缩** → "压缩到MJ/SD" 自动精简到平台最佳长度
- **批量处理** → "处理这一章" 整章小说自动拆场景批量生成
