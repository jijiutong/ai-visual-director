# 叙事结构分析器

用户丢一段小说/剧本，AI 自动拆解成三幕结构，每幕自动匹配对应的镜头组合、色彩、节奏。

## 三幕结构标准模型

```
第一幕：建制（Setup）    25%
  ↓ 展示世界观、角色、日常、引子
  ↓ 激励事件（Inciting Incident）打破平静

第二幕：对抗（Confrontation）  50%
  ↓ 冲突升级、障碍叠加、中点转折
  ↓ 最低点（All is Lost）→ 绝地反击

第三幕：解决（Resolution）   25%
  ↓ 最终对决、高潮释放、结局余韵
```

## 各幕镜头参数

### 第一幕：建制

| 参数 | 值 |
|------|-----|
| 镜头数占比 | 总镜头数的 25%（7镜中约2镜，13镜中约3镜） |
| 景别 | ELS/LS（大远景/远景）→ MS（中景） |
| 运镜 | Static / Slow Push / Wide Pan |
| 色彩 | 低对比 / 自然色 / 冷调或暖调（根据基调） |
| 灯光 | 自然光 / 柔和散射 / 日常照明 |
| 节奏 | 慢 / 稳定 / 每镜 3-5 秒 |
| 构图 | 留白 / 三分法 / 环境为主 / 引入角色 |
| 关键镜头 | 开场 establishing shot + 激励事件 close-up |

**Prompt 追加**：
```
第一幕[建制]：[镜数] 个镜头。展示世界观 [地点/时代]，引入角色 [角色名] 的日常 [日常描述]。激励事件：[事件描述] 打破平静。景别从 ELS 建立环境到 MS 引入角色。运镜 Static/Slow Push。色彩 [色调]。灯光自然光。节奏慢，每镜 3-5 秒。
```

### 第二幕：对抗

| 参数 | 值 |
|------|-----|
| 镜头数占比 | 总镜头数的 50%（7镜中约3-4镜，13镜中约6-7镜） |
| 景别 | MS/MCU → CU/ECU（逐渐逼近） |
| 运镜 | Tracking / Orbit / Push In / Handheld |
| 色彩 | 冷暖对比 / 饱和度递增 / 冲突色出现 |
| 灯光 | 分割光 / 侧光 / 局部强光 / 阴影加重 |
| 节奏 | 中→快 / 逐渐加速 / 每镜 1-3 秒 |
| 构图 | 对角线 / 过肩 / 对称对峙 / 填充画面 |
| 关键镜头 | 冲突升级序列 + 中点转折 + 最低点 |

**Prompt 追加**：
```
第二幕[对抗]：[镜数] 个镜头。冲突 [冲突描述] 逐步升级。障碍1：[障碍1]，障碍2：[障碍2]，中点转折：[转折事件]，最低点：[最低点描述]→绝地反击。景别从 MS 逐渐逼近到 CU/ECU。运镜 Tracking/Orbit/Push In/Handheld。色彩冷暖对比加剧，饱和度递增。灯光分割光+阴影加重。节奏从 3 秒加速到 1 秒。
```

### 第三幕：解决

| 参数 | 值 |
|------|-----|
| 镜头数占比 | 总镜头数的 25%（7镜中约2镜，13镜中约3镜） |
| 景别 | CU/ECU（高潮）→ LS/ELS（拉远） |
| 运镜 | Crash Zoom / Crane Up / Pull Out / Static |
| 色彩 | 极端对比（高潮）→ 统一色调（余韵） |
| 灯光 | 爆炸光/火焰光（高潮）→ 柔和光（余韵） |
| 节奏 | 极快→慢 / 高潮快切 0.5 秒 → 余韵 5 秒 |
| 构图 | 填充画面（高潮）→ 留白（余韵） |
| 关键镜头 | 最终对决 + 高潮释放 + 结局余韵 |

**Prompt 追加**：
```
第三幕[解决]：[镜数] 个镜头。最终对决：[对决描述]。高潮释放：[高潮描述]。结局：[结局描述]。余韵：[余韵描述]。景别从 CU/ECU 高潮拉远到 LS/ELS 余韵。运镜 Crash Zoom/Crane Up/Pull Out。色彩从极端对比过渡到统一色调。节奏极快（0.5秒）→ 慢（5秒）。
```

## 变体结构

### 四幕结构（韩剧/长篇）

```
第一幕 建制(20%) → 第二幕 发展(30%) → 第三幕 对抗(30%) → 第四幕 解决(20%)
```

### 倒叙结构（悬疑）

```
开头：高潮/结局片段（吸引）→ 回到起点 → 建制 → 对抗 → 回到开头补完
```

### 双线叙事（身份对立/替身）

```
A线[主线] + B线[副线/回忆] 交替进行，最终汇合
```

## 类型 → 结构自动匹配

| 故事类型 | 结构 | 说明 |
|---------|------|------|
| 动作/武侠 | 三幕标准 | 建制(展示世界)→对抗(比武升级)→解决(最终决战) |
| 爱情 | 三幕标准 | 建制(相遇)→对抗(误会/障碍)→解决(和解/在一起) |
| 悬疑 | 倒叙三幕 | 开头(案发)→回到起点→调查→反转 |
| 科幻 | 三幕标准 | 建制(发现异常)→对抗(危机升级)→解决(拯救/牺牲) |
| 成长 | 四幕结构 | 建制→发展(尝试)→对抗(挫折)→解决(成长) |
| 喜剧 | 三幕标准 | 建制(日常)→对抗(意外连连)→解决(和解/反转) |
| 悲剧 | 三幕标准 | 建制(美好)→对抗(裂痕加深)→解决(崩溃/余痛) |
| 史诗 | 三幕标准+双线 | 主线+副线交替，宏大规模 |

## 在 Prompt 中使用

在 prompt 开头追加叙事结构标注：

```
叙事结构：三幕标准。
第一幕[建制](镜1-2)：展示[世界观]，引入[角色]，激励事件[事件]。
第二幕[对抗](镜3-5)：冲突[冲突描述]升级，障碍[障碍1/障碍2]，中点转折[转折]，最低点[最低点]。
第三幕[解决](镜6-7)：最终对决[对决]，高潮[高潮]，结局[结局]，余韵[余韵]。
```
