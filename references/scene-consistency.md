# 场景一致性方法大全

7 种场景一致性锁定方法，从基础到高级，可单选可叠加。

---

| 方法 | 适合场景 | 锁定强度 | 成本 | 速度 |
|------|---------|---------|------|------|

| 2. 4 宫格 | 中小场景（房间/庭院） | ⭐⭐ | 低 | 快 |

## 一、俯拍图（Top-Down View）

鸟瞰场景全貌，从正上方垂直向下看。

**生成 prompt 模板**：
```
[场景描述] 的俯拍图，鸟瞰视角，正上方垂直向下，展示完整空间布局、地形、
建筑位置、道路走向。清晰标注各区域位置关系。
```

**优点**：空间布局一目了然，适合固定场景
**缺点**：缺少立面信息，无法看到墙壁/高度
**适用**：室内房间、庭院、城镇布局、战场地形

---

## 二、4 宫格（4-Grid）

场景 4 个关键角度拼为一张。

**生成 prompt 模板**：
```
4 宫格布局，同一[场景名称]的 4 个角度：
左上：正面全景 | 右上：侧面视角
左下：背面视角 | 右下：关键元素近景
保持光线/色调/氛围一致，禁止跳色跳光
```

**优点**：快速覆盖主要方向
**缺点**：角度有限，复杂场景不够
**适用**：中小场景（房间/庭院/走廊/街道）

---

## 三、9 宫格（9-Grid）

场景 9 个细节视角，全方位锁死。

**生成 prompt 模板**：
```
3×3 九宫格布局，同一[场景名称]的 9 个视角：
上排：正面全景 | 左侧全景 | 右侧全景
中排：入口/门 | 核心元素特写 | 道具/关键物品
下排：地面/材质 | 光源/灯光位置 | 天空/天花板
所有 9 格光线一致、色调统一、氛围连贯
```

**优点**：覆盖全面，复杂场景不遗漏
**缺点**：生成成本较高
**适用**：复杂场景（城镇/战场/宫殿/森林/洞穴）

---

## 四、全能参考模式（All-in-One Reference）

一张图同时锁死场景 + 角色 + 光照 + 道具，效率最高的锚点方案。

**生成 prompt 模板**：
```
全能参考图：[场景名称]全景环境为主体，[角色名]站立于场景中央偏左，
面向右方呈标准站姿，完整展示面部/发型/服装/配饰/武器（如有）。
场景中清晰标注：主光方向（箭头+色温）、地面材质、天空色调、
关键道具位置（红框标注）。整体氛围为[时间/天气/情绪]。
角色占画面 1/3，场景占 2/3。
```

**布局结构**：
```
┌─────────────────────────────────┐
│        场景全景环境              │
│  ☀ 主光方向 →                   │
│    ┌────┐                       │
│    │角色│  地面材质              │
│    │站姿│  ▓▓▓▓▓▓▓▓            │
│    └────┘                       │
│         □ 关键道具（红框标注）    │
│         天空色调：xxxx            │
└─────────────────────────────────┘
```

**优点**：
- 一张图锁定全部变量，Seedance 只需 1 张参考图 + prompt
- 场景/角色/光照/道具同框，AI 自动学会关联关系
- 适合快速验证、成本最低

**缺点**：
- 角色细节不如独立角色卡丰富
- 换场景需重新生成

**适用**：快速出片、单场景视频、Seedance 简略模式

**触发命令**：回复「全能参考图」或「all-in-one」

## 五、720° 全景图（720° Panorama）

生成或合成一张 360°×180° 的全景环境图，覆盖所有可能拍摄方向。

**方式 A：直接生成（需支持全景的模型）**
```
全景图：[场景名称] equirectangular panorama，360 degree，
覆盖东西南北四个方向所有视觉信息，统一光照和色调，
天空完整、地面连续、无接缝
```

**方式 B：多角度合成（无全景模型时）**
- 生成 6 个方向：前/后/左/右/上/下（cubemap 方式）
- 拼接为一张全景图
- 输入 Seedance 作为全局环境参考

**方式 C：实景拍摄（如有条件）**
- 用手机全景模式或 360° 相机拍摄
- 作为参考图直接输入

**触发命令**：回复「生成全景图」或「720全景」

**优点**：覆盖所有角度，视频中无论镜头转到哪个方向都有参考
**缺点**：生成/拼接耗时，大文件

## 六、环绕视频截图法（Orbit Video Screenshot）

先生成一个环绕场景的短视频（或用 3D 引擎渲染），再从中截取关键帧作为多角度参考。

**步骤**：
```
1. 用 AI 视频工具生成一段「环绕场景旋转」的短片（5s 即可）
   prompt：「camera orbits 360° around [scene]，steady rotation，bird eye view slowly descending to eye level」

2. 截取 6-8 张关键帧（每 45°~60° 一张）

3. 拼接为一张参考图（按角度排列），或单独作为多张参考图输入
```

**使用 Seedance 生成环绕视频**：
```
prompt：[场景名称]，camera orbits 360 degrees around the scene，
steady slow rotation，no speed change，maintain constant distance，
展示场景完整空间结构
```

**截取角度建议**：
| 角度 | 说明 |
|------|------|
| 0° | 正面 |
| 45° | 右前 |
| 90° | 右侧 |
| 135° | 右后 |
| 180° | 背面 |
| 225° | 左后 |
| 270° | 左侧 |
| 315° | 左前 |

**优点**：真实连续空间参考，最接近实拍效果
**缺点**：需两轮生成（先视频后截图），耗时加倍
**适用**：3D 引擎场景、实景拍摄、对空间一致性要求极高的项目

**触发命令**：回复「环绕截图」或「视频截图法」

## 七、GPT Image 2 镜头角度修改（Camera Angle Remix）

利用 GPT Image 2 的图像编辑/生成能力，基于一张已有的场景参考图，直接生成不同镜头角度的分镜图。

**核心思路**：不重新生成场景，而是让 AI 理解现有场景的空间结构，从不同角度「拍摄」，保证场景一致性。

**步骤**：
```
1. 先生成一张场景参考图（俯拍图/全景环境图）
   → prompt：「[场景描述]，全景环境，展示完整空间和布局，光线明亮清晰」

2. 基于这张场景图，逐镜生成不同角度的分镜图：
   → 镜 1：「同一场景，从[门口]看向[室内]，低角度仰拍，突出空间高度」
   → 镜 2：「同一场景，从[窗口]看向[角色位置]，侧面中景，柔光逆光」
   → 镜 3：「同一场景，从[天花板]角落俯拍，广角，覆盖整个空间」
   → 镜 N：「同一场景，[角色]的 POV 视角，看向[目标方向]」
```

**关键约束**（每镜 prompt 必须包含）：
```
同一场景，所有家具/建筑/地形/光源位置与参考图完全一致，
仅改变拍摄位置和镜头角度。禁止改变场景布局。
光线方向/色调/氛围与参考图一致。
```

**优点**：
- 直接出分镜图，不需要额外视频工具
- 每镜都是独立高质量的图
- 场景一致性天然有保证（因为每镜都引用同一场景参考）

**缺点**：
- 依赖 GPT Image 2 的场景理解能力
- 复杂场景可能出不一致的角度
- 每次修改角度是一张新图，不是实时渲染

**触发命令**：回复「镜头角度分镜」或「camera remix」

**与其他方法配合**：
```
最强组合：720° 全景图 + GPT Image 2 镜头角度修改
1. 先生成 720° 全景图（锁定全局空间）
2. 再逐镜从全景图中「取景」，生成各角度分镜图
3. 得到 100% 场景一致的 N 张分镜图
```

## 方法叠加策略

| 预算 / 要求 | 推荐组合 |
|------------|---------|
| 快速验证 | 全能参考模式（一张搞定） |
| 标准短片（15s） | 俯拍图 + 角色卡 |
| 正式成片（30s） | 9 宫格 + 角色卡 + 光照材质图 |
| 复杂大场景 | 720° 全景图 + 环绕截图法 |
| 最强一致性 | 720° 全景图 + GPT Image 2 镜头角度分镜 |
| 极致性价比 | 全能参考模式 + GPT Image 2 镜头角度分镜 |
