# 锁定状态

记录用户已确认并锁定的内容。任何引擎在任何生成前必须先读此文件。锁定字段优先级最高，LLM 不得擅自修改。

---

## 一、锁定优先级

```
锁定字段 > 用户本轮明确指定 > 用户模糊描述 > 知识库建议 > LLM 自由发挥
```

锁定字段 = 铁律。即使用户本轮说"稍微改一下"，也必须确认是临时变化还是解锁重设。

## 三、锁定触发词

用户出现以下表达时，系统必须进入锁定判断：

```text
定了
锁定
就这版
这个脸定了
这个角色定了
这个场景定了
这个风格定了
```

### 部分锁定

允许只锁一部分字段，不要求整套资产全部锁死。

```text
脸型定了，服装可以再试。
发型锁定，表情可以变化。
场景结构定了，光线可以再试。
角色 DNA 锁定，构图可以继续变化。
```

示例：

```json
{
  "character.face_dna": "locked",
  "character.costume_core": "draft",
  "scene.spatial_structure": "locked",
  "lighting.mood": "draft"
}
```

### 解锁触发词

```text
解锁
解除锁定
全部解锁
重新来
这个可以改了
```

`全部解锁` 必须二次确认，防止误操作：

```text
你正在解除全部锁定，会允许角色/场景/格式被重新生成。请确认。
```

## 四、锁定生效规则

### 1. 锁定字段不可被任何引擎自动修改

锁定字段不可被任何引擎自动修改，包括：

```text
story-intake
video-director
asset-plan
consistency-trigger
prompt-qc
auto-repair
prompt-declutter
multi-version
style-migration
```

`auto-repair` 只能修格式、缺项、引用、字数、噪点等问题，不得重写 locked 字段。

### 2. 锁定字段优先于知识库

当知识库建议与锁定字段冲突时：

```text
丢弃知识库建议
保留锁定字段
只在可变字段内补充创意
```

示例：

```text
锁定：角色黑色短发
知识库建议：仙侠女主适合白色长发
处理：丢弃白色长发建议，保持黑色短发
```

### 3. 多版本必须尊重锁定

`multi-version`、`fusion`、`style-migration` 等发散能力必须把 locked 字段作为不可变输入。

允许变化：

```text
构图
光线
表情
姿态
景别
色彩氛围
```

不允许变化：

```text
面部 DNA
服装核心
发型
武器/道具
场景空间结构
资产用途
```

## 五、状态生命周期

| 状态 | 含义 | 写回权限 |
|------|------|----------|
| `draft` | 本轮生成草稿，默认状态 | 可被引擎继续修，但不覆盖主状态 |
| `derived` | 派生探索版本，用于多版本/风格迁移/发散 | 不污染主状态 |
| `locked` | 用户明确锁定的字段 | 不可被自动修改 |
| `committed` | 用户确认采用并写回项目 | 写入 `state/variable-registry.md` 和 `projects/` |

默认所有新生成内容都是 `draft`。

`derived` 只用于派生探索，不污染主状态。用户明确 `/commit` 后，才允许从 `derived` 写入 `committed`。

`committed` 必须写入：

```text
state/variable-registry.md
projects/<project_id>/state/
state/state-commit-log.md
```

---

## 六、每次生成前必须执行

每次生成前必须执行以下步骤：

```text
1. 读取 state/lock-state.md
2. 提取所有非 "—" 的锁定字段
3. 形成不可变字段列表
4. 在 prompt/template/engine 决策中声明这些字段保持不变
5. 输出后检查输出是否违反锁定字段
```

如果违反：

```text
阻断输出
进入 /repair
只修冲突字段
不得重做未冲突部分
```

## 七、联动

| 模块 | 如何使用 lock-state |
|------|--------------------|
| `task-router` | 每次路由前读取锁定状态，决定是否需要 command-gate 限制 |
| `command-gate` | 根据 lock-state 生成 must_use / may_use / forbidden_use 和 write_policy |
| `auto-repair` | 修复时避开 locked 字段，只修允许修改的字段 |
| `variable-registry` | committed 内容写回后同步主变量 |
| `state-commit-log` | 记录锁定、解锁、commit、rollback 历史 |
| `prompt-qc` | 检查输出是否违反 locked 字段 |
| `asset-qc` | 检查资产用途锁定是否被误用 |
