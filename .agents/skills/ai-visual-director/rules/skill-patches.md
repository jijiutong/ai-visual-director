## R001 科幻题材不得默认机甲化

**触发条件**：故事含太空港/仿生人/休眠舱/空间站/木星/外星/AI/星际，但未出现机甲/驾驶舱/外骨骼/战斗装甲/机器人战斗单位。

**错误倾向**：系统自动推荐「科幻机甲」风格。

**正确行为**：
- 太空/仿生人/AI → 暗黑科幻
- 赛博/未来都市 → 赛博佛学
- 有机甲/驾驶舱/外骨骼关键词 → 科幻机甲

**禁止**：不得在无机甲关键词时推荐科幻机甲。不得在仿生人故事里自动加入巨型机器人。

**发现于**：T03-A 废弃太空港仿生少女
**修复文件**：storyboard/SKILL.md、ai-visual-director/SKILL.md（推荐引擎表拆分）

## R002 古代场景禁止现代光源词

**触发条件**：故事背景为古代/玄幻/历史/武侠（中国古风、西方中世纪、日本江户等），用户请求进行王家卫/韩国犯罪/都市情绪等现代风格迁移。

**错误倾向**：风格迁移时出现「霓虹灯光」「霓虹灯牌」「LED」「街灯」「城市灯光」「路灯」等现代物理光源描述。

**正确行为**：现代感通过以下方式模拟：
- 允许词：`霓虹感` `霓虹感调色` `绿色色罩` `湿润光晕` `胶片颗粒` `水面反光` `慢门拖影` `抽帧` `降格慢镜` `偏斜构图`
- 禁用词：`霓虹灯光` `霓虹灯牌` `霓虹灯` `LED` `LED光源` `城市灯光` `街灯` `现代灯源` `电子屏幕` `广告牌灯光` `路灯`

**核心原则**：霓虹感 = 调色模拟 ≠ 真的有灯

**发现于**：TM01 东方玄幻→王家卫风格迁移
**修复文件**：ai-visual-director/SKILL.md（执行规则 + 冲突检测表）

## R003 Prompt 禁止生成可读文字

**触发条件**：画面中出现屏幕、标识、书本、纸张、广告牌等可能承载文字的表面。

**错误倾向**：Prompt 中写「屏幕显示「提交成功」」「写着 404」「弹出 Error」等要求模型生成具体文字的描述。

**正确行为**：全部替换为纯视觉描述。

| ❌ 不可用 | ✅ 替代 |
|-----------|--------|
| 屏幕显示「提交成功」 | 屏幕弹出一个绿色勾选图标，柔和提示光在屏幕上跳动 |
| 「Error」「404」 | 红色闪烁提示光 + 警告状态指示色块 |
| 任何可读中文/英文/数字 | 色块/图标/光影条纹/模糊字符纹理 |

**强制扫描**：Prompt 输出前逐句扫描：
- 引号内中文 `「...」` → 检查是否屏幕/书本文字
- 引号内英文 `"... "` → 同上
- 数字序列 → 替换为颜色+光影状态
- `显示`/`写着`/`出现`/`弹出` + 具体文字 → 替换为图标/光影/色块
- 出现一次扣 2 分，扫描不通过不输出

**发现于**：R04 蓝色桌面宠物（帧6 提交成功）
**修复文件**：storyboard/SKILL.md、ai-visual-director/SKILL.md

## R004 角色卡默认禁止文字标注

**触发条件**：对所有角色使用 `/character` 生成角色设定卡。

**错误倾向**：角色卡中出现「红色标注不可变细节」+ 负面词含 `no text overlay`，二者冲突。

**正确行为**：
- 出图模式（默认）：所有细节通过光影、对焦、构图对比传达，不叠加文字/箭头/编号/方框/引线
- 导演标注版（仅当用户说「导演标注版」「标注」时）：允许红色标注/编号/文字说明

**发现于**：/character 墨渊角色卡
**修复文件**：character/SKILL.md（拆分为出图模式 + 导演标注版）、ai-visual-director/SKILL.md

## R005 现实都市狭小空间运镜限制

**触发条件**：故事场景为便利店/办公室/车内/卧室/电梯/走廊等单间面积<20平米的室内。

**错误倾向**：出现「环绕慢旋」「中轴线旋转穿越」「航拍级上升」「穿越墙体」等不可物理执行的运镜。

**正确行为**：仅允许以下运镜：
- 固定机位
- 缓慢前推/后拉
- 水平横移
- 低角度仰拍
- 过肩镜头
- 浅景深变焦

**发现于**：R02 深夜便利店（帧5 极慢旋转推进）
**修复文件**：storyboard/SKILL.md、ai-visual-director/SKILL.md

## R006 默认推荐使用通用风格名而非品牌名

**触发条件**：故事含桌面宠物/童话/绘本/治愈/手绘/日系日常等元素。

**错误倾向**：推荐名直接使用「Disney Pixar 3D」「Studio Ghibli」「吉卜力」「迪士尼」等品牌名。

**正确行为**：
- `3D动画电影风`（替代 Disney/Pixar）
- `日系手绘治愈风`（替代 吉卜力/Ghibli）
- `童话绘本风`（替代 Disney 2D）
- 品牌名仅在用户说「看全部」或明确指定时展示

**发现于**：R04 蓝色桌面宠物
**修复文件**：storyboard/SKILL.md、ai-visual-director/SKILL.md（风格名规范）

## R007 角色身份冲突必须输出解释

**触发条件**：角色描述含高位身份词（第一剑修/宗师/剑尊/掌门/前辈/老祖/天下第一/绝世/天尊/剑圣/武帝），但在故事中为后辈/徒弟/儿子等身份。

**错误倾向**：直接强行改写用户描述（如把「第一剑修」改成「天才学徒」），或默认给过低年龄。

**正确行为**：输出【身份解释】块，说明适配方案（天才型青年/陨落后重修/年轻一代榜首等），保留用户原始描述，给用户切换选项。

**触发词**：`第一剑修` `宗师` `剑尊` `掌门` `前辈` `老祖` `第一` `天下第一` `绝世` `天尊` `剑圣` `武帝`

**发现于**：/character 墨渊
**修复文件**：character/SKILL.md、ai-visual-director/SKILL.md

## R008 一键生成不得自动融合风格

**触发条件**：用户输入 `/storyboard 一键生成 [故事]`

**错误倾向**：系统自动使用风格融合（如「东方玄幻 + 王家卫情绪融合」）。

**正确行为**：一键生成默认使用单一最匹配主风格。只有用户明确说「融合」「多版本」「A+B」时才触发融合。

**发现于**：R01 雨夜古寺（早期版本自动融合了王家卫）
**修复文件**：storyboard/SKILL.md、ai-visual-director/SKILL.md

## R009 风格迁移必须显性声明锁定范围

**触发条件**：用户说「换成X风格但保持角色」。

**错误倾向**：迁移后剧情事实改变（剧情/关系/冲突结果/镜头数/时长/关键动作）。

**正确行为**：每次迁移必须开头声明：
```
🔒 锁定不变：角色DNA / 剧情事实 / 关键动作 / 镜数时长
🎨 变更维度：色彩/光影/构图/运镜/节奏/表演方式
```

**发现于**：TM01 东方玄幻→王家卫
**修复文件**：ai-visual-director/SKILL.md

---

## R010 单镜修改必须保留时间码 + 连续字段自动补桥

**触发条件**：用户说「第X镜 [修改内容]」，涉及天气/时间/地点/角色外观等连续性强字段。

**错误倾向**：时间码漂移，或天气修改不处理前后镜衔接。

**正确行为**：
- 严格保留镜号、原时间码、原镜头顺序
- 天气变更：在修改镜的前一镜末尾插入过渡元素
- 联动微调前后镜相关元素
- 输出连续性提示说明影响范围

**发现于**：第5镜雨改雪（早期版本未处理过渡）
**修复文件**：ai-visual-director/SKILL.md
