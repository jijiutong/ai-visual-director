# 批量故事处理

一次丢整章小说，自动拆分成多个场景/回合，批量生成对应故事板。

## 拆分规则

### 按场景拆分

AI 自动识别故事中的场景切换点：

```
原文：
"他在办公室加班到凌晨，雨已经下了三个小时。
（场景切换）
走到天台，城市的霓虹在雨中模糊成一片。
（场景切换）
手机屏幕亮了，是她的消息。
（场景切换）
他看着屏幕，雨打在脸上，分不清是雨水还是泪水。"

拆分结果：
场景1：办公室加班（内景/夜晚/雨）
场景2：天台俯瞰城市（外景/夜晚/雨/霓虹）
场景3：手机消息特写（近景/屏幕光/雨）
场景4：雨中情绪（外景/夜晚/雨/情感高潮）
```

### 按回合拆分（打斗/对话）

```
回合1：相遇/挑衅
回合2：第一招
回合3：反击
回合4：高潮对决
回合5：结局
```

### 按章节拆分

用户说"处理第1-3章"时，每章生成一个独立故事板。

## 批量输出格式

```
【批量处理 - 《[片名]》第 X 章】

共拆分为 [N] 个场景：

【场景 1：[场景名]】
位置：[原文段落]
风格：[推荐风格]
格式：[推荐格式]
```[prompt]```

【场景 2：[场景名]】
...
```

## 场景关系标注

每个场景标注与前后场景的关系：

```
场景1 → 场景2：时间推进（加班→天台），情绪升级（疲惫→孤独）
场景2 → 场景3：动作连续（看手机→读消息），情绪转折（孤独→期待）
场景3 → 场景4：情绪爆发（读消息→泪崩），色彩从冷到暖
```

## 用户指令

| 用户指令 | 行为 |
|---------|------|
| "处理这一章" | 自动拆分场景 + 逐个生成 |
| "处理第1-3章" | 每章独立拆分 + 批量生成 |
| "只看拆分" | 仅输出场景拆分结果，不生成 prompt |
| "场景X用风格Y" | 为指定场景指定风格 |
| "全部统一风格Y" | 所有章节用同一风格 |

## 在 Prompt 中使用

批量输出时标注：

```
【场景 X/Y】《[片名]》- [场景名]
位置：原文第 [X] 段
关系：← [前场景] → [后场景]
```
